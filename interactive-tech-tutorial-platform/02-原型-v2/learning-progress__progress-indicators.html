进度指示器（嵌入式规范） 继续学习提示与入口

本页是嵌入式 UI 规范，不是独立路由页面

「进度指示器」定义步骤导航进度条、章节目录完成勾选、教程包整体完成度三类组件在 学习工作台 与 主题课程库 页面中的实际呈现方式与数据来源。下方每个区块均可交互，用于演示状态变化逻辑；数据模型遵循 D4「不拦人」范式——完成判定不校验代码正确性，只以是否点击「下一步」经过该步骤为准。

区块一 · 学习工作台底部步骤导航进度条

数据字段:currentStepId totalSteps

来源模块:learning-workspace

在实际页面中该组件吸底固定于学习工作台视口底部，高度 64px。此处以卡片形式演示同一组件在「前进 / 后退」时的实时行为，可点击下方按钮体验。

第 3 步 / 共 15 步 状态提升（Lifting State Up）
当前步（实心） 已完成（描边） 未开始（空心灰）
点击「下一步」时：当前步骤先被写入 completedStepIds（乐观更新，本地即时生效，不等待任何异步确认），随后 currentStepId 前移一位，进度条与「第 X 步」文案随之刷新。点击「上一步」仅回退 currentStepId，不会撤销已标记的完成状态。

区块二 · 章节目录树节点完成标记

数据字段:completedStepIds currentStepId chapterCompletionRate

来源模块:learning-workspace / theme-library

点击任意步骤行可跳转「回看」——完成标记不代表锁定，跳转不会撤销已有完成状态。

目录跳转（无论前跳或后跳）仅更新 currentStepId，不追溯补全被跳过步骤的完成标记，避免「跳着看」被误判为学完。

关键数据字段汇总

供内容维护者与开发对齐口径

字段名 类型 说明
currentStepIdstring当前所在步骤，用于进度条「第 X 步」文案与目录树高亮定位
completedStepIdsstring[]已标记完成的步骤集合，去重、只增不减
totalStepsnumber教程包/章节步骤总数，来自 TutorialPackage 配置
chapterCompletionRatenumber (0-100)章节完成率，向下取整整数百分比
tutorialCompletionRatenumber (0-100)教程包整体完成率，驱动课程库卡片百分比/环形/条形展示

边界态：localStorage 不可用时的降级展示

当次会话内存态（不报错、不阻断）

隐私模式/浏览器禁用本地存储时，进度条与目录树在当前会话内仍正常显示；但刷新或重新打开后，续学提示与课程库完成度不生效，卡片按钮回退为「开始学习」。
