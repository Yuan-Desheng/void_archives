## 三、偏差①：waitFor 在移动端沙箱死灰复燃 🔴

`code-sandbox__sandbox-mobile-tab.html` 第 354 行：
```html
<div id="waitForBar" class="waitfor-bar pending">
  <span>等待你在「代码」Tab 里把 count 初始值改成 10…</span>
```
状态机 `pending → success`，用户改对了才转绿。

**这就是 D4 明令砍掉的 R4 等待机制**——「等待用户完成特定操作后再继续」。
AI 在移动端沙箱把它重新造了出来，还顺带把老聚光灯的琥珀色 `#F0A63B` 带了回来。

- 范围：**仅此 1 页**，主工作台 / 桌面沙箱面板均无
- 定性：违反 D4，属 AI「自动补全」了需求外功能（规范 §五② 要删/标的对象）
- **待 Owner 定**：删掉这个 waitForBar，还是承认「沙箱内的轻量引导」不算 R4、予以保留？
  （二者不同：R4 是全局阻塞式教程步进；这个是沙箱内单点提示。但用词和状态机同源，容易滑坡）

### 三.1 处理（2026-07-19）✅
Owner 决策：**删掉**。已从 `code-sandbox__sandbox-mobile-tab.html` 移除：
- `#waitForBar` DOM + `.waitfor-bar` CSS + `#codeTabBadge` 琥珀角标
- `renderWaitForBar()` / `renderCodeTabBadge()` / `checkWaitFor()` 三个函数及全部调用

## 四、偏差②：配色非纯灰阶，是政务蓝 ⚠️

D5 要求「shadcn neutral 纯灰阶零彩色」。实测 token：

| token | 值 | 是否灰阶 |
|---|---|---|
| `--color-bg` | `#F5F7FA` | ✅ 中性 slate |
| `--color-surface` | `#FFFFFF` | ✅ |
| `--color-text` | `#1E293B` | ✅ slate |
| `--color-border` | `#E2E8F0` | ✅ |
| **`--color-primary`** | **`#3B5BDB`** | ❌ **蓝色** |
| **`--color-accent`** | **`#0EA5B7`** | ❌ **青色** |

即：底座是中性灰，但主色/强调色是彩色——**styleKey=gov（政务蓝）而非 shadcn neutral**。
（注：语法高亮里的大量彩色 hex 是代码着色，合理，不计入。）

- 定性：D5 部分未落实。比 v1 深色仪表盘已大幅收敛，但没到「纯灰阶」。

### 四.4 处理（2026-07-19）✅
Owner 决策：**严格拉成 shadcn 纯中性灰**。
发现配色来自**三套源**，逐一处理：
1. CSS 变量 `--color-primary/accent` → slate（亮 `#1E293B`/`#475569`，暗 `#64748B`/`#94A3B8`）
2. **Tailwind 调色板** `primary{}`/`accent{}`（真正大头，`bg-primary-500` 用了 47 次）→ 整条换成 slate 阶梯
3. 硬编码品牌色 15 处（`#3B5BDB`/`#0EA5B7`/`rgba(59,91,219)`）→ slate
- **语法高亮 `.tok-*` 的彩色刻意保留**（代码高亮就该有颜色，Catppuccin 那套原样不动）——
  用 `NN: '#hex'`（调色板专属写法）作为替换判别，精准避开 `color: #hex`（语法高亮）
- 实测：`bg-primary-500` 渲染为 `rgb(100,116,139)` slate；无蓝青主色残留；绿色状态点（语义色）保留

### 四.5 附带修复：pre-existing 崩溃 bug ✅
修配色时发现 `mobile-tab` 页**每次加载抛 `ReferenceError: Cannot access 'currentFiles' before initialization`**。
- 根因：`applyTheme()` 在顶层早调用 → `updateStatusPanel()` 访问 `currentFiles`，而后者是 `let`、在更下方才声明（TDZ）
- 验证是 **AI 生成时就存在**（git HEAD 原版同样 87ms 抛错），非本次编辑引入
- 修复：把主题初始化调用挪到 `currentFiles` 声明之后的 init 块
- 实测：0 console error
- 归档意义：正是规范 §五② 要抓的「AI 产出看着对≠对」——18 页里这页从没被点开测过，静态看不出
