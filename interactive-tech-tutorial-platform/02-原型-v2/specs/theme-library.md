# 主题课程库（theme-library）功能定义

> 对应模块总览：`key: theme-library` — 展示可学主题列表与切换，呈现每个 TutorialPackage 的章节结构，承载 R8 多主题可扩展。
> 依据：`01-需求/原始需求.md`、`02-原型-v2/specs/00-需求提取与范围界定.md`（D4 范式转向后版本）

## 1. 模块概述

「主题课程库」是学习者进入互动教程系统的**入口与目录层**，只做两件事，不做别的：

1. **主题包（TutorialPackage）列表与切换** —— 以卡片形式展示平台已配置的所有教程包（首个为 React，预留 Vue / JavaScript / 算法），每张卡片展示标题、简介、章节数、总步数，以及基于本地进度的完成度提示；点击进入即切到「学习工作台」（`learning-workspace` 模块）继续该包的学习。
2. **章节 / 步骤大纲呈现** —— 进入某个教程包后，以树形大纲展示该包 `chapters[].steps[]` 的完整结构，标注当前学习到的位置与已完成步骤，支持点击任意步骤**直接跳转**——本模块不做内容渲染、不做代码沙箱，跳转后由「学习工作台」接管。

**架构原则（R8 验收核心，本模块必须体现）**：主题列表与大纲的渲染必须完全由 `TutorialPackage` 配置数组 / 树遍历驱动，代码里不得出现针对某个具体主题（如「React」「Vue」）的 if/switch 分支。验收标准——工程侧在配置层新增一个 `vue.json`，本模块无需改一行代码即可在主题选择页、章节大纲里正确渲染出 Vue 主题的卡片与目录树。

**范式边界（D4 已定，不得违反）**：本模块延续「左讲解 + 右沙箱，靠内容引导、不拦人」的范式——大纲中的步骤**不设锁定/解锁状态**，任意步骤在任意时刻都可以点击直接跳转，没有 R3/R4/R5 那套「必须完成上一步才能进入下一步」的强制序列。已完成标记只是**信息展示**，不是门禁。

**与相邻模块的边界**：
- 进度数据（当前步、已完成步骤集合）由 `learning-progress` 模块负责读写 localStorage，本模块只**读取并展示**，不拥有进度状态。
- 点击某个步骤后的 MDX 讲解渲染、代码沙箱，由 `learning-workspace` / `code-sandbox` 模块负责，本模块只负责导航跳转前的目录呈现。
- 视觉主题（灰阶配色、亮暗切换）由 `appearance-theme` 模块统一提供，本模块直接复用其组件与样式变量。

## 2. 用户故事

| # | 角色 | 故事 |
|---|------|------|
| 1 | 学习者 | 我想打开平台就看到有哪些教程主题、每个主题讲什么、多少章节，以便挑一个开始学 |
| 2 | 学习者 | 我想在某个主题里看到完整的章节/步骤大纲，一眼知道这个主题一共要学多少内容 |
| 3 | 学习者 | 我上次学到某一步，今天回来想直接看到「继续学习」并跳回上次的位置，不用从头翻 |
| 4 | 学习者 | 我想在大纲里跳到任意一步复习或提前预览后面的内容，不希望被要求"先做完上一步" |
| 5 | 学习者 | 我在 React 主题学到一半，想切到 Vue 主题看看，两边进度应该互不影响，回来时 React 还在原来的位置 |
| 6 | 学习者 | 步骤很多（比如 Hooks 章节 17 篇、Router 章节 24 篇）时，我想只看当前章节展开、其余折叠，或者按关键字快速定位 |
| 7 | 内容维护者 | 我新增一个 `vue.json` 配置后，希望它原样出现在主题选择页和章节大纲里，不需要求助研发改代码 |
| 8 | 内容维护者 | 我想在调试内容时打开大纲，快速核对章节顺序、步骤数量是否和我配置的一致 |

## 3. 功能点清单

### 3.1 主题包列表与切换
- F1 主题包卡片列表：以卡片形式展示所有 `TutorialPackage`（标题、简介、章节数、总步数、完成度百分比）
- F2 主题包状态标识：`active`（可进入）/ `maintenance`（维护中）/ `draft`（内容筹备中，章节数为 0），后两者卡片置灰不可点击
- F3 「继续学习」/「开始学习」CTA：本地已有进度的包，卡片显示「继续学习」，点击直达上次的 `currentStepId`；无进度的包显示「开始学习」，直达第一章第一步
- F4 主题包完成度提示：卡片上展示「已完成步骤数 / 总步数」及百分比小进度条（数据来自 `learning-progress`，本模块只读）
- F5 主题间独立切换：学习者可随时从任一教程包返回主题选择页，切换到另一个包；各包进度互不覆盖、互不影响

### 3.2 章节 / 步骤大纲
- F6 大纲树渲染：按 `TutorialPackage.chapters[].steps[]` 顺序渲染两级结构（章节 → 步骤），支持章节折叠 / 展开
- F7 步骤状态图标（仅信息展示，非门禁）：已完成（对勾）、当前所在（高亮）、未学过（空心圆）——三态之间不存在"锁定"态
- F8 自由跳转：点击大纲中**任意**章节下的**任意**步骤，立即跳转到该步骤（交由 `learning-workspace` 渲染），不做"是否解锁"判断
- F9 章节级统计：每章节标题旁展示「已完成步骤数 / 该章节总步数」
- F10 当前位置定位：打开大纲时自动展开「当前学习到的章节」并滚动定位到当前步骤，其余章节默认折叠
- F11 大纲内检索：步骤数较多的教程包（如 React 的 Hooks/Router 章节）在大纲顶部提供关键字快速过滤，输入即高亮/收窄匹配的步骤
- F12 返回主题库入口：大纲内提供明显入口，一键返回主题选择页

## 4. 业务规则

1. **数据驱动约束**：主题选择页与章节大纲的渲染逻辑必须完全由 `TutorialPackage` 配置数组 / 树遍历驱动，代码中不得出现针对某个具体主题 id 的 if/switch 分支；新增一个符合 Schema 的 `TutorialPackage` 配置，两处界面均需零改动正确渲染（R8 验收口径）。
2. **不设锁定态**：延续 D4「不拦人」范式，大纲中所有步骤在任意时刻均可直接点击跳转，不存在"必须完成上一步才解锁下一步"的强制规则；已完成状态仅用于视觉提示，不影响可点击性。
3. **状态置灰规则**：`status = maintenance` 或 `chapters` 为空数组（`draft`）的教程包卡片禁用点击，并给出对应文案提示；不计入"可学习主题"统计。
4. **进度只读**：本模块不写入 `UserProgress`（当前步、已完成步骤集合），只从 `learning-progress` 模块暴露的 localStorage 数据中读取用于展示完成度与「继续学习」定位；实际的步骤完成判定与写入时机由 `learning-workspace` 负责。
5. **包间进度隔离**：每个 `packageId` 的进度数据独立存储，切换主题包不清空、不覆盖彼此的 `currentStepId` 与 `completedStepIds`。
6. **顺序以配置为准**：章节与步骤的展示顺序严格按配置中的声明顺序（数组顺序或显式 `order` 字段），不做字母序或其他隐式排序。
7. **完成度计算**：章节完成度 = 该章节 `completedStepIds` 命中的步骤数 / 该章节总步数；包完成度 = 全部已完成步骤数 / 全部步骤数；实时反映最新本地进度，无需等待网络（原型阶段全程本地计算，无服务端往返）。

## 5. 页面清单

| 页面 | 职责 |
|------|------|
| 主题选择页 | 平台入口，卡片形式展示所有教程包（标题/简介/章节数/完成度/状态），承载主题切换、继续学习/开始学习入口 |
| 章节大纲面板 | 单个教程包内的目录导航层：树形展示章节/步骤、标注当前位置与完成状态、支持自由跳转与关键字检索；可作为进入学习工作台前的独立视图，也可作为工作台内的可展开侧栏/抽屉复用 |

## 6. 关键数据字段

### TutorialPackage（教程包配置，本模块只读消费）
| 字段 | 说明 |
|------|------|
| id | 唯一标识，字符串类型（不枚举限定），如 `react` / `vue` |
| meta.title / meta.description | 主题标题、简介 |
| meta.language | 沙箱语言模式（供 `code-sandbox` 消费，本模块仅展示用途） |
| status | `active` / `maintenance` / `draft` |
| chapters[] | 章节数组，每章含 id、title、steps[] |
| chapters[].steps[] | 步骤数组，每步含 id、title（大纲只需 id/title，讲解正文与代码由 `learning-workspace`/`code-sandbox` 消费） |
| totalSteps | 总步数（可由 chapters 派生，用于卡片展示） |

### UserProgress（学习进度，来自 learning-progress 模块，本模块只读）
| 字段 | 说明 |
|------|------|
| packageId | 关联的教程包 id |
| completedStepIds[] | 已完成步骤 id 列表 |
| currentStepId | 当前 / 最近学习到的步骤（"继续学习"跳转锚点） |

### 大纲节点（前端渲染态，由上述两者派生，不落库）
| 字段 | 说明 |
|------|------|
| nodeType | `chapter` \| `step` |
| displayState | `done`（已完成）\| `current`（当前）\| `idle`（未学过）——无 `locked` 态 |
| progressPercent | 章节级 / 包级完成百分比 |

## 7. 异常 / 边界

| 场景 | 处理方式 |
|------|---------|
| 教程包 `chapters` 为空数组（内容筹备中） | 卡片置灰、禁用点击，标注「内容筹备中」 |
| 教程包 `status = maintenance` | 卡片置灰、禁用点击，标注「维护中」 |
| 章节存在但 `steps` 为空数组 | 大纲中该章节显示「暂无内容」，不影响其余章节的展开与跳转 |
| 步骤 id 重复或大纲结构成环（配置错误） | 开发环境下抛出明显的配置校验提示；生产环境降级为只渲染首次出现的节点，跳过重复项，不崩溃 |
| localStorage 中无该包的进度记录 | 视为从零开始，卡片显示「开始学习」，大纲无高亮当前位置，不报错 |
| localStorage 被清空 / 不可用（隐私模式等） | 完成度与「继续学习」均按"无进度"兜底展示，不阻塞主题选择与大纲浏览 |
| 步骤数量很大（如 Hooks 17 篇、Router 24 篇） | 大纲默认只展开"当前所在章节"，其余折叠；顶部提供关键字过滤，避免整页超长滚动 |
| 375px 窄屏下的大纲 | 收纳为可展开的抽屉，不常驻挤压主内容区，避免横向滚动（对齐 R7） |
| 新增教程包缺少 `meta.description` 等非必填展示字段 | 卡片对应位置留空或显示占位符，不影响卡片可点击性 |
| 学习工作台中途切换主题包 | 当前包的 `currentStepId` 已由 `learning-progress` 逐步保存，切换不丢失，返回时大纲自动定位到离开前的位置 |
