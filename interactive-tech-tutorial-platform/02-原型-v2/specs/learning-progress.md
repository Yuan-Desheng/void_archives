# 模块功能定义：学习进度（learning-progress）

## 0. 模块概述

「学习进度」负责本平台**进度指示的呈现**与**进度数据的原型级记忆**两件事：

1. **进度指示**：把学习者当前所在的位置、已完成的步骤/章节，转化为「当前步/总步数」文案、进度条、目录树打勾标记、教程包完成度百分比等可视化元素，嵌入到「学习工作台」「主题课程库」等其他模块的页面中。
2. **原型级记忆**：以浏览器 `localStorage` 记录当前步骤与已完成步骤集合，支持用户重新打开教程包时「继续学习」而非从头开始。

本模块不负责教程内容渲染（属 learning-workspace）、不负责代码沙箱运行（属 code-sandbox）、不负责主题/课程包结构展示本身（属 theme-library），只消费其他模块在「步骤前进/跳转」时产生的导航事件，把进度**写入本地**并**可视化**。

**范式对齐（D4）**：本平台已砍掉「高亮/等待机制/聚光灯」（R3/R4/R5），忠于 Vue3「不拦人」范式。因此本模块**不做 `waitFor` 强制完成校验**——步骤是否「完成」由学习者是否前进经过该步骤决定，而非某种代码正确性判定，绝不阻断学习者的自主节奏。

**范围边界（对齐阶段①范围界定）**：本项目原型阶段**不做用户系统/登录**、**不做跨端同步**、**不落库（PostgreSQL）**。进度数据只存在于当前浏览器的 `localStorage`，是单人单设备的原型级记忆，不是生产级账户体系的一部分。这些能力若未来需要，属于开发阶段的后端范畴，明确不在本模块原型职责内。

核心定位：
- 对**学习者**：打开教程后无需任何登录/操作即可开始学习；进度自动、静默地被记住；重新打开教程包时可选择「继续学习」或「从头开始」。
- 对**内容维护者**：本模块的完成率统计口径（已完成步骤数/总步骤数）需与 TutorialPackage 的章节/步骤结构保持一致，供内容组织参考。
- 对**Owner**：本模块无独立管理界面，进度数据不对外暴露、不导出、不分析。

## 1. 用户故事

1. 作为学习者，我打开一个教程包后可以立刻开始学习，不需要登录、不需要任何前置操作。
2. 作为学习者，我完成当前步骤并点击「下一步」时，希望看到「当前步/总步数」的进度条随之推进（如 `3 / 15`），这样我清楚自己学到哪、还剩多少。
3. 作为学习者，我在章节目录树里，希望一眼看出哪些步骤已经学过（打勾）、当前在哪一步（高亮）、哪些还没开始（默认态）。
4. 作为学习者，我关闭浏览器标签页，几天后重新打开同一个教程包，希望看到「继续学习（第 X 章 · 步骤 Y）」的提示，点一下就能回到上次的位置，而不用从头翻找。
5. 作为学习者，我如果就是想从头复习，也希望能方便地选择「从头开始」，而不是被迫接续上次进度。
6. 作为学习者，我在主题课程库浏览多个教程包（如 React、Vue）时，希望在每个教程包的卡片上看到自己在该主题下的完成度百分比，以及「继续学习」还是「开始学习」的按钮文案（视是否已有进度而定）。
7. 作为学习者，我在目录树里往回跳转查看已经学过的内容，希望这个「回看」动作不会把我已完成的进度撤销或打乱。
8. 作为学习者，我如果清空浏览器数据或换了一台设备，能接受进度从头开始——这是本地记忆工具的合理代价，只要产品不因此报错或崩溃。
9. 作为学习者，我把一个教程包全部学完后，希望目录树/课程库卡片明确显示「已完成」，按钮文案变为「重新学习」而非继续显示未完成态。
10. 作为学习者，我误触「从头开始」时不希望进度被无声清空，希望有一次简单的二次确认。

## 2. 功能点清单

| 编号 | 功能点 | 说明 |
|---|---|---|
| F1 | 当前步记录 | 学习者每次进入某一步骤（前进/后退/目录跳转），记录 `currentStepId`，无论该步骤是否被判定为完成，用于断点续学 |
| F2 | 步骤完成判定 | 不做 `waitFor` 强制校验（D4）；学习者点击「下一步」离开某步骤时，该步骤即标记为 `completed`；目录跳转不追溯标记跳过的步骤 |
| F3 | 已完成步骤集合 | `completedStepIds` 去重存储；只增不减——已完成状态不因用户回看/倒退而被撤销，仅整体重置（F9）才清空 |
| F4 | 完成率计算 | 章节完成率 = 该章节已完成步骤数 / 章节总步骤数；教程包完成率 = Σ各章节已完成步骤数 / Σ各章节总步骤数 |
| F5 | 步骤导航进度指示 | 学习工作台底部「当前步/总步数」文案与进度条（对标 Vue3 `1 / 15` 形态），随 F1/F2 实时更新 |
| F6 | 目录树完成标记 | 章节/步骤目录中三态展示：已完成（打勾）、当前所在（高亮）、未开始（默认态） |
| F7 | 继续学习提示 | 重新打开某教程包时，若本地存在 `currentStepId` 且非第一步，展示「继续学习（章节 X · 步骤 Y）」与「从头开始」两个入口，不强制跳转、不自动覆盖 |
| F8 | 课程库卡片完成度 | 主题课程库中每个教程包卡片展示整体完成度百分比，按钮文案随是否有进度切换为「继续学习」/「开始学习」/「重新学习」 |
| F9 | 进度重置 | 提供「从头开始」/「重新学习」操作，二次确认后清空该 `tutorialId` 对应的本地进度记录，不影响其他教程包 |
| F10 | 多主题进度隔离 | 不同教程包（tutorialId）的进度各自独立存储于 localStorage，互不覆盖、互不干扰 |

## 3. 业务规则

### 3.1 存储结构
- 统一以对象形式存储于 localStorage，键名如 `ittp:progress`，值结构为 `{ [tutorialId]: TutorialProgress }`，按 `tutorialId` 隔离（F10）。
- 单条 `TutorialProgress` 至少包含：`currentStepId`、`completedStepIds`（数组）、`lastVisitedAt`。
- 本模块不引入 `userId` / `guestId` 等身份字段——原型阶段无登录，进度天然与「当前浏览器」绑定，无需额外匿名标识。

### 3.2 完成判定（对齐 D4「不拦人」）
- 步骤完成 = 学习者点击「下一步」使该步骤成为「已经过」的步骤，不校验代码是否正确、不判断沙箱运行结果。
- 目录树直接跳转到某步骤（无论前跳或后跳）仅更新 `currentStepId`，不追溯补全跳过步骤的完成标记——避免「跳着看」却被误标记为完成的假象。
- 已标记完成的步骤，学习者随后跳转回看不会撤销其完成状态（只增不减），除非执行 F9 整体重置。

### 3.3 断点续学
- `currentStepId` 在每次进入步骤时更新，与 `completedStepIds` 是两个独立字段，分别服务于「续学定位」与「完成率计算」。
- 用户重新进入某教程包时：若存在 `currentStepId` 且不是第一步，展示 F7 提示；用户可选择「继续学习」（跳转至 `currentStepId`）或「从头开始」（触发 F9 重置后进入第一步）。
- 首次访问某教程包（无任何本地记录）时不展示续学提示，直接进入第一章第一步。

### 3.4 完成率与展示
- 完成率计算结果向下取整到整数百分比展示（如 66%）。
- 教程包全部步骤完成时，目录树与课程库卡片进入「已完成」态，卡片按钮文案变为「重新学习」。

### 3.5 边界与降级
- localStorage 不可用（隐私模式、浏览器禁用、容量超限）时，进度记忆功能整体降级为「当次会话内存态」：F5/F6 的当前会话内指示仍正常工作，但刷新/重新打开后 F7 续学提示与 F8 完成度不生效，课程库卡片按钮回退为「开始学习」——不报错、不阻断学习流程。
- 教程包结构（章节/步骤）发生变更后，历史 `completedStepIds`/`currentStepId` 中已不存在的 stepId 不参与当前完成率计算、不触发续学提示，静默忽略即可，不报错。
- 多个浏览器标签同时打开同一教程包时，采用「最后写入者生效」，不做锁机制——内部原型工具可接受轻量数据竞争，不引入额外复杂度。

## 4. 页面清单

| 页面 key | 页面名称 | 职责概述 |
|---|---|---|
| progress-indicators | 进度指示器（嵌入式规范） | 非独立路由页面，定义步骤导航进度条、目录树完成勾选、教程包整体完成度在其他模块页面中的呈现规范与数据来源 |
| continue-learning | 继续学习提示与入口 | 定义用户重新进入教程包时「继续学习/从头开始」的提示交互，以及课程库卡片上完成度与按钮文案的驱动逻辑 |

> 说明：本模块两个页面均为**嵌入式 UI 规范**，不是独立可跳转的路由页面，而是「学习进度」相关组件在 learning-workspace（学习工作台）、theme-library（主题课程库）等其他模块页面中的呈现与交互规范。单列是为了让设计/开发有明确的交付对象；实现阶段如需合并调整，不影响功能范围。

### 4.1 progress-indicators（进度指示器）
- **职责**：定义步骤完成后，目录树/进度条/课程库卡片应如何呈现「已完成 / 进行中 / 未开始」与完成度百分比。
- **主要 UI 区块**：
  - 学习工作台底部步骤导航进度条（当前步/总步数文本 + 进度条，如 `3 / 15`）
  - 章节目录树节点完成标记（打勾图标，区分「已完成 / 当前所在 / 未开始」三态）
  - 教程包整体完成度展示（用于课程库卡片的百分比/环形或条形指示）
- **关键数据字段**：`currentStepId`、`completedStepIds`、`totalSteps`、`chapterCompletionRate`、`tutorialCompletionRate`
- **关键交互**：步骤完成（点击「下一步」）时目录树节点即时打勾（本地写入后乐观更新，不等待任何服务端确认）；点击已完成步骤仍可跳转回看内容（完成标记不代表锁定）；进度条随步骤前进/后退动态更新当前步数字。

### 4.2 continue-learning（继续学习提示与入口）
- **职责**：学习者重新进入某教程包时，基于本地记录判断展示「继续学习」或「从头开始」，并驱动课程库卡片的按钮文案与跳转目标。
- **主要 UI 区块**：
  - 续学横幅/轻提示（进入教程包时展示：「继续学习（第 X 章 · 步骤 Y）」+「从头开始」两个按钮）
  - 课程库教程包卡片内的完成度百分比 + 「继续学习 / 开始学习 / 重新学习」按钮（三态文案随进度切换）
  - 「从头开始」二次确认提示（清空该教程包进度前的简单确认，防误触）
- **关键数据字段**：`tutorialId`、`currentStepId`、`tutorialCompletionRate`、`lastVisitedAt`
- **关键交互**：点击「继续学习」→ 跳转至 learning-workspace 对应 `currentStepId`；点击「从头开始」→ 二次确认后触发 F9 重置该教程包进度并跳转第一步；课程库卡片点击「继续学习/开始学习」→ 直达断点位置或第一步。

## 5. 关键数据字段汇总

| 字段名 | 类型 | 说明 |
|---|---|---|
| tutorialId | string | 教程包唯一标识，用于隔离不同主题的进度存储 |
| chapterId | string | 章节标识，用于章节完成率计算与目录树分组 |
| stepId | string | 步骤标识 |
| currentStepId | string | 该教程包下用户当前所在步骤，用于断点续学定位 |
| completedStepIds | string[] | 该教程包下已标记完成的步骤集合（去重，只增不减） |
| stepStatus | enum: completed / current / not_started | 目录树节点展示态，由 currentStepId / completedStepIds 派生 |
| totalSteps | number | 该教程包/章节的步骤总数，来自 TutorialPackage 配置 |
| chapterCompletionRate | number (0-100) | 章节完成率，向下取整为整数百分比 |
| tutorialCompletionRate | number (0-100) | 教程包整体完成率 |
| lastVisitedAt | timestamp | 最后一次进入该教程包的时间，用于续学提示排序/展示 |

## 6. 异常与边界场景

| 场景 | 处理方式 |
|---|---|
| localStorage 不可用（隐私模式/浏览器禁用/容量超限） | 降级为当次会话内存态：当前会话内进度指示正常，但刷新/重新打开后续学提示与课程库完成度不生效，卡片回退为「开始学习」；不报错、不阻断学习 |
| 教程包结构变更导致历史 stepId 失效 | 该 stepId 不参与当前完成率计算，不触发续学提示；静默忽略，不报错、不阻断学习 |
| 用户在目录树中直接跳转到较后的步骤 | 仅更新 currentStepId，不追溯补全中间步骤为已完成，避免「跳着看」被误判为学完 |
| 用户回看已完成的步骤 | 完成标记不被撤销，允许自由回看，不锁定、不弹出任何限制提示 |
| 多个浏览器标签同时打开同一教程包 | 以「最后写入者生效」处理，不加锁；内部原型工具接受轻量数据竞争 |
| 用户清空浏览器数据/更换设备 | 视为全新学习者，回到「从头开始」默认态，不报错、不提示数据丢失 |
| 首次访问某教程包（无任何本地记录） | 不展示继续学习提示，直接进入第一章第一步 |
| 教程包全部步骤已完成 | 目录树/课程库卡片进入「已完成」态，按钮文案变为「重新学习」 |
| 「从头开始」被误触 | 触发前需一次简单二次确认，防止无声清空已有进度 |
| localStorage 数据被手动删除 | 视为全新学习者，不报错，回到默认态 |
