# 功能定义：学习工作台（learning-workspace）

> 依据：`02-原型-v2/specs/00-需求提取与范围界定.md`（阶段②，2026-07-17 范式转向 D4/D5 之后版本）
> 范式基线：忠于 Vue3 官方互动教程（cn.vuejs.org/tutorial）实测结果——**左讲解 + 右沙箱，靠内容引导，全程不拦人**。本模块及全部子项**明令不得**出现高亮定位（R3）/等待阻塞（R4）/聚光灯遮罩（R5），这三项已于 D4 被 Owner 拍板砍掉。

---

## 0. 模块概述

`learning-workspace` 是整个平台**唯一的核心学习页**，对标 Vue3 官方教程实测到的骨架：左侧 `.vt-doc`（讲解）+ 右侧 `vue-repl`（split-pane 沙箱）+ 底部「当前步 / 总步数」步骤导航。它是学习者进入某个教程包后，从第一步学到最后一步全程停留的唯一界面。

**本模块承载的职责（做实，不做虚）**：

1. **R1 内容展示**：渲染当前 step 的 MDX/Markdown 讲解正文（含代码块语法高亮），承载在左侧讲解区。
2. **R2 步骤导航**：底部「上一步 / 下一步」按钮 + 「当前步 / 总步数」进度指示，驱动 step 前进/后退，无任何阻塞条件（对标 Vue3「下一步」按钮永远可点）。
3. **三段式布局壳**：顶部信息条（章节/步骤标题、返回主题库入口）+ 左讲解区 + 右沙箱容器区 + 底部导航条，是三个兄弟模块（code-sandbox / learning-progress / appearance-theme）的**宿主**，负责把它们编排进这一张页面，而非重新实现它们各自的逻辑。
4. **响应式布局切换**：375 / 768 / 1440 三档下，workspace 自身的分栏/单栏切换与移动端「讲解 ⇄ 沙箱」Tab 切换（视觉 token 由 appearance-theme 提供，但切换控件本身是 workspace 布局的直接体现，故落在本模块）。

**不属于本模块职责（明确划清，避免范围蔓延）**：

- 不实现 Sandpack 编辑器 / 多文件切换 / Run / 预览刷新 / 「给我看答案」的具体逻辑——这些是 **code-sandbox 模块**（R6/R10）的职责，workspace 只把整块沙箱面板作为子组件挂载到右侧区域。
- 不实现学习进度的存储与读取逻辑（localStorage 读写、断点续学计算）——这是 **learning-progress 模块**的职责，workspace 只在步骤切换时发出「进入了某 step」的事件，并在进入页面时读取「上次停留位置」作为初始 step。
- 不实现主题包列表、切换、章节目录树浏览——这是 **theme-library 模块**（R8）的职责，workspace 假定「已选定的 TutorialPackage + 当前 chapterId/stepId」已经作为输入传入。
- 不实现亮 / 暗主题切换开关本身与配色 token 定义——这是 **appearance-theme 模块**（R9）的职责，workspace 只应用其提供的样式变量。
- 不做高亮定位 / 聚光灯遮罩 / waitFor 阻塞（R3/R4/R5）——D4 已明令砍掉，任何设计稿或代码中出现这三者视为范式违规。

## 1. 用户故事

| # | 角色 | 故事 |
|---|---|---|
| U1 | 学习者 | 作为学习者，我进入某个教程包后，希望左边看到当前步骤的讲解文字（含代码块高亮），右边看到一个真的能编辑代码的沙箱，两块同时可见，不用来回切页面。 |
| U2 | 学习者 | 作为学习者，我希望页面底部清楚告诉我「现在第几步 / 一共几步」（如 `3 / 15`），心里有数还要学多久。 |
| U3 | 学习者 | 作为学习者，我希望点「下一步」永远能点（除非已经是最后一步），不会因为我"没做什么操作"就被卡住——学不学得会是我自己判断，不是页面替我判断。 |
| U4 | 学习者 | 作为学习者，我希望点「上一步」能随时回看前面的内容，且不会丢失我在当前步骤沙箱里已经改动的代码（在同一次会话内）。 |
| U5 | 学习者 | 作为学习者，我希望走到某一章的最后一步、点「下一步」时，能自然过渡到下一章的第一步，不用我自己回目录再点进去。 |
| U6 | 学习者 | 作为学习者，我希望走到全教程包最后一步时，「下一步」按钮明确告诉我"已经是最后一步"，而不是变灰却不说明原因，或者点了没反应。 |
| U7 | 学习者（手机端） | 作为用手机学习的人，我希望讲解和沙箱不会挤在一屏里看不清，能一键切换"看讲解"还是"写代码"，默认先看讲解。 |
| U8 | 学习者 | 作为学习者，我希望重新打开这个教程包时，直接回到我上次停留的那一步，而不是又从第一步开始。 |
| U9 | 内容维护者 | 作为内容维护者，我希望只要我的 `TutorialPackage` JSON 符合 Schema（`chapters[].steps[].description/files/solution`），workspace 就能正确渲染，不需要我为了适配页面再改一遍内容结构。 |
| U10 | Owner | 作为 Owner，我希望验证「内容引擎分离」时，只看 workspace 代码就能确认：这里没有任何一行写死某个具体主题（React/Vue）的文案或逻辑。 |

## 2. 功能点清单

| 编号 | 功能点 | 说明 |
|---|---|---|
| F1 | 顶部信息条 | 展示当前教程包标题、当前章节标题、返回主题库（theme-library）的入口链接 |
| F2 | 左侧讲解区渲染 | 渲染当前 step 的 `description`（MDX/Markdown），支持标题、列表、图片、代码块语法高亮、内嵌提示块 |
| F3 | 右侧沙箱容器 | 挂载 code-sandbox 模块提供的沙箱面板组件，向其传入当前 step 的 `files`（初始代码）；不实现面板内部逻辑 |
| F4 | 底部步骤导航条 | 「上一步」「下一步」按钮 + 「当前步 / 总步数」进度指示（如 `3 / 15`），总步数为**整个教程包跨章节的全局步数**，与 Vue3 实测行为一致 |
| F5 | 步骤前进/后退 | 点击「下一步」推进到下一个 step（跨章节自动衔接）；点击「上一步」回退到上一个 step；两者**均不设阻塞条件**，随时可点（首/末步边界见业务规则） |
| F6 | 初始定位 | 页面加载时向 learning-progress 模块请求「该教程包上次停留的 step」，作为初始展示位置；无历史记录则从第一章第一步开始 |
| F7 | 步骤切换事件上报 | 每次 step 变化（含前进/后退/深链跳转）向 learning-progress 模块发出「当前 step」事件，由该模块决定是否落地存储 |
| F8 | 响应式三段式布局 | 1440：左讲解 + 右沙箱左右并排；768：左右并排（讲解区收窄）或视设计决定上下分栏；375：讲解 / 沙箱单栏 + Tab 切换，默认展示讲解 |
| F9 | 移动端讲解/沙箱切换 | 375 断点下提供「讲解」「代码」两个 Tab（或等效切换控件），切换不丢失沙箱当前编辑内容 |
| F10 | 深链定位 | 支持通过 URL 直接定位到教程包内的某个 chapterId/stepId（用于 theme-library 目录树跳转、浏览器前进/后退、分享链接） |

## 3. 业务规则

### 3.1 步骤推进与边界（核心：不拦人）
1. 「下一步」按钮**永远可点**，唯一例外是**当前已是整个教程包的最后一步**——此时按钮呈禁用态并展示明确文案（如「已经是最后一步」），不得只变灰不说明原因。
2. 「上一步」按钮**永远可点**，唯一例外是**当前已是第一步**——此时按钮禁用，无需额外文案（首步语义自明）。
3. 推进/后退**不依赖任何操作检测**（不检测是否点击过沙箱内某元素、是否输入过代码）——这是与旧范式（waitFor）的根本区别，必须在实现和设计评审中重点核查。
4. 章节边界自动衔接：某章节最后一个 step 点「下一步」→ 自动进入下一章节第一个 step；顶部信息条的「当前章节标题」随之更新。

### 3.2 进度指示口径
5. 「当前步 / 总步数」采用**全局连续计数**（跨章节累加），不是「当前章节内的步数」，与 Vue3 实测的 `1 / 15` 一致，避免学习者误以为"每章都要重新数到 1"。
6. 章节标题在顶部信息条单独展示，弥补"全局计数看不出章节切换点"的信息缺口。

### 3.3 讲解内容渲染
7. `step.description` 按 MDX/Markdown 规则渲染；代码块按 fenced code block 的语言标注做语法高亮，高亮配色随亮/暗主题切换（token 由 appearance-theme 提供，本模块只消费）。
8. 讲解区不解析、不执行内容中的可交互脚本（避免内容维护者写入的 Markdown 携带脚本引发安全或渲染副作用），仅做安全的富文本渲染。

### 3.4 沙箱联动（仅编排，不越界实现）
9. 进入某 step 时，workspace 把该 step 的 `files` 字段作为初始代码传给 code-sandbox 面板；`files` 为空/未定义时，沙箱面板使用教程包 `meta.defaultCode` 兜底（该兜底逻辑属于 code-sandbox 模块内部行为，workspace 只负责把 `defaultCode` 一并传下去）。
10. 用户在当前 step 沙箱内的编辑内容，在**同一次会话内**、`上一步`/`下一步` 来回切换时是否保留，由 code-sandbox 模块自行决定（workspace 不做任何"离开即丢弃/离开即保留"的判定，只负责触发 step 切换本身）。

### 3.5 响应式规则
11. 1440 断点：左右两栏固定分栏，不出现横向滚动条。
12. 768 断点：布局收窄但仍保持左右两区域同屏可见（不强制切换为单栏），除非设计阶段评估后确认 768 下内容过挤才降级为切换模式——原型阶段默认保持双栏。
13. 375 断点：讲解区与沙箱区**不同屏并列**，通过 Tab（或等效控件）切换，默认展示讲解区；切换控件本身需在三档断点下均可达（不得只在某一断点下才出现）。

### 3.6 深链与跳转
14. 深链（URL 直达某 chapterId/stepId）优先于 learning-progress 的"上次停留位置"——若 URL 携带明确的 step 定位，以 URL 为准；无 URL 定位时才回退到读取历史进度。
15. 深链指向的 chapterId/stepId 在当前 TutorialPackage 中不存在时，视为无效深链，回退到该教程包第一章第一步，并（由上层路由/提示层）给出"该步骤不存在"的轻量提示。

## 4. 页面清单（每页职责）

| 页面 key | 页面名称 | 职责 |
|---|---|---|
| `workspace` | 学习工作台（核心页，含三档响应式状态） | 唯一核心页面：顶部信息条 + 左侧 MDX 讲解区 + 右侧沙箱容器（挂载 code-sandbox 组件）+ 底部步骤导航条（上一步/下一步 + 当前步/总步数）；一个页面覆盖 375/768/1440 三档响应式布局与移动端讲解/沙箱切换，不拆分为多个独立页面 |

> 说明：本模块刻意只定义**一个页面**——它是"内容与引擎分离"架构下唯一需要被反复验证的宿主页，主题选择、进度总览、沙箱内部编辑器等均已在其余四个模块（theme-library / code-sandbox / learning-progress / appearance-theme）中定义，不在此重复列出，避免同一页面职责被拆到多份 spec 里失焦。

## 5. 关键数据字段

### 5.1 TutorialPackage（教程包，workspace 的输入契约，D4 后的新 Schema）
| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 教程包唯一标识（不再枚举限定，保证 R8 新增主题无需改引擎） |
| `meta.title` | string | 主题标题 |
| `meta.description` | string | 主题简介 |
| `meta.language` | string | 沙箱语言模式（如 `jsx` / `vue` / `js`），透传给 code-sandbox |
| `meta.defaultCode` | string | 沙箱默认代码模板（`files` 未提供时的兜底） |
| `chapters` | Chapter[] | 章节列表 |

### 5.2 Chapter（章节）
| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 章节唯一标识 |
| `title` | string | 章节标题（顶部信息条展示） |
| `steps` | Step[] | 步骤列表 |

### 5.3 Step（步骤，workspace 渲染的核心驱动单元）
| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 步骤唯一标识 |
| `title` | string | 步骤标题 |
| `description` | string（MDX/Markdown） | 左侧讲解正文 |
| `files` | Record<string, string> \| undefined | 该步初始代码（多文件），传递给 code-sandbox，本模块不解析其内容 |
| `solution` | Record<string, string> \| undefined | 答案代码，本模块不消费（由 code-sandbox 的「给我看答案」使用），仅作为透传字段随 step 一并存在 |

> 注：`selector` / `waitFor` 字段已于 D4 删除，workspace 的 Step 结构中不应出现，也不读取。

### 5.4 WorkspaceRuntimeState（运行时状态，非配置数据）
| 字段 | 类型 | 说明 |
|---|---|---|
| `currentChapterId` / `currentStepId` | string | 当前所在位置 |
| `globalStepIndex` / `totalStepCount` | number | 底部进度指示的分子/分母（跨章节全局计数） |
| `activeMobilePane` | enum `explain` \| `sandbox` | 375 断点下当前展示的是讲解区还是沙箱区 |

### 5.5 与相邻模块的接口字段（workspace 发起，非本模块存储）
| 字段 | 方向 | 说明 |
|---|---|---|
| `lastVisitedStepId` | 读（来自 learning-progress） | 页面加载时用于初始定位 |
| `stepEnteredEvent{ packageId, chapterId, stepId, timestamp }` | 写（发往 learning-progress） | 每次 step 变化时上报 |
| `sandboxInitialFiles` | 写（发往 code-sandbox） | 当前 step 的 `files`（或兜底 `meta.defaultCode`） |

## 6. 异常 / 边界场景

| 场景 | 处理方式 |
|---|---|
| 当前已是整个教程包最后一步，用户点"下一步" | 按钮禁用并展示明确文案（如"已经是最后一步"），不触发任何跳转（规则 1） |
| 当前已是第一步，用户点"上一步" | 按钮禁用，无需额外文案（规则 2） |
| URL 深链指向的 chapterId/stepId 在当前教程包中不存在 | 回退到第一章第一步，并给出"该步骤不存在"提示（规则 15） |
| TutorialPackage 的某个 step 缺少 `description` | 讲解区渲染为空态提示（如"本步骤暂无讲解内容"），不影响该 step 的沙箱区正常挂载，不阻断整体渲染 |
| `files` 与 `meta.defaultCode` 均为空 | 沙箱区展示空白/最小模板（具体兜底逻辑属 code-sandbox 模块，workspace 只保证不因此崩溃） |
| learning-progress 模块无法返回"上次停留位置"（如首次访问、接口异常） | 默认从第一章第一步开始，不阻塞页面渲染 |
| 章节内 `steps` 为空数组 | 该章节在全局步数计数中不产生任何 step，"下一步"越过该章节直接进入下一个有内容的章节；若这是最后一个有内容的章节之后，视为教程包已结束 |
| 用户在 375 断点下正处于"代码"Tab，此时点"下一步"切到下一 step | 保持当前激活的 Tab（若用户在看代码就切到下一步的代码，而非强制跳回讲解 Tab） |
| 快速连续点击"下一步"（如连点触发多次跳转） | 前端做基本的点击去抖/状态锁定，避免一次点击触发多次 step 跳转导致计数错乱 |
| 浏览器前进/后退按钮 | 与深链跳转走同一套定位逻辑（规则 14），保持 URL 与 workspace 展示状态一致 |
| 教程包整体不存在或已下线（由上层路由判定） | 不在本模块处理范围内——由 theme-library / 路由层处理"教程不存在"空态，workspace 假定被挂载时输入的 TutorialPackage 一定有效 |

## 7. 验收关注点（供设计/开发阶段参考，非本文档强制范围）

- 375 / 768 / 1440 三档下左右分栏与移动端 Tab 切换均无横向滚动、无内容重叠。
- 明暗双主题下讲解区文字、代码高亮对比度达 WCAG AA。
- 全教程包首尾步骤的按钮禁用态文案清晰、无"假死"观感。
- 章节边界衔接（最后一步→下一章第一步）过渡自然，顶部章节标题正确同步更新。
- 走查代码/设计稿确认全程无 selector 高亮、无聚光灯遮罩、无 waitFor 阻塞逻辑残留（D4 红线）。
