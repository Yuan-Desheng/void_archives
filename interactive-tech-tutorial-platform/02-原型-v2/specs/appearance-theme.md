# 功能定义：外观与主题（appearance-theme）

> 模块 key：`appearance-theme`
> 承载需求：R7 响应式、R9 外观主题
> 上游真源：`prototype/specs/00-需求提取与范围界定.md`

---

## 一、模块概述

本模块不是一个独立的业务功能页，而是**贯穿全站的视觉与布局基础设施**：

1. **外观（R9）**：统一采用 shadcn/ui 的 neutral 纯灰阶配色体系 + Inter 字体 + `0.5rem` 圆角，提供亮色 / 暗色两套主题，并支持一键切换、跟随系统、以及切换记忆。视觉基调克制、工程化，**明令禁止仪表盘式高饱和撞色**（历史决策 D5 已废弃 `#4C6FFF / #22D3EE / #FDBA33` 配色方案，不得复现）。
2. **响应式（R7）**：定义 375 / 768 / 1440 三档断点下，全站（尤其是核心页「学习工作台」的左讲解 + 右沙箱双栏布局）如何重排，确保任何断点下**无横向滚动、无内容重叠遮挡**；移动端（<768px）下讲解区与沙箱区从「并排」收缩为「二选一可切换」的单栏形态。

本模块的产出物是**设计令牌（design tokens）+ 全局交互组件（主题切换器、断点响应规则、移动端讲解/沙箱切换 Tab）**，被其他所有模块（theme-library、learning-workspace、code-sandbox、learning-progress）复用，本身不承载独立业务数据。

## 二、用户故事

| 角色 | 故事 |
|---|---|
| 学习者 | 作为学习者，我希望能一键切换亮色/暗色主题，这样晚上学习时不会被强光刺眼。 |
| 学习者 | 作为学习者，我希望首次打开时主题能跟随我系统的深色模式设置，不用每次手动调。 |
| 学习者 | 作为学习者，我希望下次打开页面时，上次选的主题还在，不用重新设置。 |
| 学习者 | 作为用手机/平板学习的学习者，我希望在小屏上能方便地在「看讲解」和「写代码」之间切换，而不是被挤成两条看不清的窄栏。 |
| 学习者 | 作为学习者，无论用什么设备打开，我都不希望出现左右滚动条或内容被裁切/重叠。 |
| 内容维护者 | 作为内容维护者，我希望所有页面共用同一套灰阶色板和字号规范，不用为每个新页面单独调色，保证视觉一致。 |
| Owner | 作为 Owner，我要确保任何页面的文字对比度都达到 WCAG AA 标准，不会出现"浅灰配浅灰"看不清的问题。 |

## 三、功能点清单

### R9 外观主题

| 编号 | 功能点 | 说明 |
|---|---|---|
| F1 | 亮/暗双主题切换器 | 全局导航栏固定位置的主题切换控件，支持「浅色 / 深色 / 跟随系统」三态切换 |
| F2 | shadcn neutral 灰阶设计令牌 | 统一的背景/前景/边框/强调色变量（`--background` `--foreground` `--border` `--primary` 等），亮暗两套取值 |
| F3 | 字体与圆角规范 | 全局采用 Inter 字体（中文回退系统字体栈），组件统一 `radius: 0.5rem` |
| F4 | 主题记忆 | 用户选择的主题模式存入 `localStorage`，刷新/重新打开后保持 |
| F5 | 系统主题跟随 | 选择「跟随系统」时监听 `prefers-color-scheme`，系统切换时页面实时联动 |
| F6 | 代码高亮双主题适配 | Sandpack 编辑器与 MDX 代码块的语法高亮方案随亮/暗主题联动切换（不出现暗主题下刺眼的浅色代码块） |
| F7 | 对比度校验基线 | 所有文本/背景组合需满足 WCAG AA（正文对比度 ≥ 4.5:1，大字/图标 ≥ 3:1） |

### R7 响应式

| 编号 | 功能点 | 说明 |
|---|---|---|
| F8 | 三档断点定义 | `375px`（移动端）/ `768px`（平板）/ `1440px`（桌面端）为核心验证宽度，采用流式断点（非仅三个精确像素点） |
| F9 | 桌面端双栏布局（≥1024px） | 学习工作台维持「左讲解 + 右沙箱」左右并排，可拖拽或固定比例分栏 |
| F10 | 平板端布局（768–1023px） | 双栏保留但收窄，或视内容密度降级为可切换单栏（由具体页面模块决定，本模块提供切换能力） |
| F11 | 移动端讲解/沙箱切换（<768px） | 讲解区与沙箱区不再并排，收缩为单栏堆叠，顶部提供「讲解 / 沙箱」Tab 切换，一次只显示一个区域 |
| F12 | 无横向滚动保证 | 三档断点下页面主体（含代码块、Sandpack 面板、表格）均不产生 body 级横向滚动条；确需横向滚动的内容（如长代码行）在组件内部单独滚动 |
| F13 | 导航/侧边栏响应式收纳 | 章节目录侧边栏在移动端收起为抽屉/汉堡菜单，桌面端常驻展开 |
| F14 | 触达尺寸适配 | 移动端可点击元素（Tab、按钮、切换器）满足最小 44×44px 触达面积 |

## 四、业务规则

1. **主题模式优先级**：`localStorage` 中已有用户显式选择的主题 → 用其值；无记录时 → 读取 `prefers-color-scheme` 作为初始值；用户任何时候手动切换即写回 `localStorage` 并覆盖系统跟随（除非用户显式选回「跟随系统」）。
2. **禁止仪表盘配色**：本模块产出的配色令牌只允许 shadcn neutral 灰阶体系 + 极少量强调色（用于 primary 按钮/链接/焦点态），不得引入高饱和多色渐变或荧光色（对应决策 D5，红线，不可因"好看"擅自扩展）。
3. **移动端讲解/沙箱互斥显示规则**：<768px 时，讲解区与沙箱区为互斥 Tab，不允许同时半屏显示导致两者都过窄不可用；切换 Tab 不丢失当前编辑器内的代码改动（沙箱状态保持在内存中）。
4. **断点判定为响应式而非专有设备判断**：以视口宽度（viewport width）做媒体查询判断，不做 UA 嗅探；三档宽度为验收基准点，实现需保证中间宽度（如 500px、900px）同样不破版。
5. **主题切换不刷新页面**：切换亮/暗主题为纯前端状态切换（CSS 变量/类名切换），不触发整页 reload，不丢失当前学习进度或沙箱代码。
6. **代码高亮主题联动**：Sandpack 编辑器主题与 MDX 代码块高亮主题必须与页面亮/暗主题保持一致，不允许出现"页面暗色、代码块仍浅色背景"的不一致态。
7. **对比度不达标即视为缺陷**：任何新增页面/组件如配色导致对比度低于 WCAG AA 阈值，视为本模块验收不通过项，需回归调色而非加粗/加下划线掩盖。
8. **响应式规则全局生效**：本模块定义的断点与切换规则对 theme-library、learning-workspace、code-sandbox、learning-progress 四个模块的所有页面统一生效，不允许某个页面自定义一套不同的断点值。

## 五、页面清单

本模块无独立的业务主页面，其能力以「全局组件 + 设置面板 + 响应式示例页」形式体现，供其他模块页面复用与校验。原型阶段落地为以下 2 个可点击验证的页面/面板：

| key | 页面名 | 职责 |
|---|---|---|
| theme-switcher-panel | 外观设置面板 | 承载亮/暗/跟随系统主题切换器，展示当前生效的灰阶设计令牌样例（颜色板、字体、圆角、按钮/文本对比度示例），供用户与内容维护者验证 R9 效果 |
| responsive-showcase | 响应式与移动端切换示例页 | 以「学习工作台」典型内容为样例，模拟 375/768/1440 三档断点下的布局表现，并演示移动端讲解区/沙箱区 Tab 切换交互，供验证 R7 效果 |

> 说明：`theme-switcher-panel` 中的主题切换控件在实际产品中会以「全局导航栏右上角图标按钮」的形式嵌入每个页面，本页面是其独立可测试的载体；`responsive-showcase` 同理是断点规则的独立验证载体，实际断点规则应用于 learning-workspace 等真实页面。

## 六、关键数据字段

### 6.1 主题偏好（localStorage，key: `theme-preference`）

| 字段 | 类型 | 说明 |
|---|---|---|
| `mode` | `'light' \| 'dark' \| 'system'` | 用户选择的主题模式，默认 `'system'` |
| `resolvedTheme` | `'light' \| 'dark'` | 当前实际生效的主题（`mode='system'` 时由系统偏好解析得出，非持久化字段，运行时计算） |
| `updatedAt` | `string`（ISO 时间戳） | 最近一次用户手动切换主题的时间，用于原型阶段调试展示 |

### 6.2 设计令牌（design tokens，随主题模式变化的一组 CSS 变量，非用户数据，仅列关键项）

| 令牌 | 亮色取值（示意） | 暗色取值（示意） | 用途 |
|---|---|---|---|
| `--background` | `oklch(1 0 0)` 近纯白 | `oklch(0.145 0 0)` 近纯黑灰 | 页面/卡片背景 |
| `--foreground` | `oklch(0.145 0 0)` 深灰 | `oklch(0.985 0 0)` 近白 | 正文文字 |
| `--border` | 浅灰 | 深灰 | 分隔线、卡片边框 |
| `--primary` | 深灰/近黑 | 近白 | 主按钮、强调态（非彩色） |
| `--muted-foreground` | 中灰 | 中灰（略提亮） | 次要文字、说明性文字 |
| `--radius` | `0.5rem`（亮暗一致） | `0.5rem`（亮暗一致） | 组件圆角 |

### 6.3 响应式断点定义

| 断点名 | 宽度阈值 | 对应布局形态 |
|---|---|---|
| `mobile` | `< 768px` | 讲解/沙箱单栏 + Tab 切换，侧边栏收起为抽屉 |
| `tablet` | `768px – 1023px` | 双栏收窄或可切换（视具体页面），侧边栏可收起 |
| `desktop` | `≥ 1024px` | 双栏并排，侧边栏常驻 |

### 6.4 移动端讲解/沙箱切换状态（页面内运行时状态，非持久化）

| 字段 | 类型 | 说明 |
|---|---|---|
| `activeMobilePanel` | `'explain' \| 'sandbox'` | 当前移动端展示的面板，默认 `'explain'` |

## 七、异常 / 边界

| 场景 | 处理方式 |
|---|---|
| 浏览器不支持 `prefers-color-scheme` 媒体查询 | 视为系统偏好不可知，默认回退到 `light` |
| `localStorage` 不可用（隐私模式/禁用存储） | 主题切换仍在当前会话内生效（内存态），仅不做跨刷新持久化，不报错阻断使用 |
| 视口宽度恰好落在断点边界（如 767px/768px） | 以 `< 768px` 视为 mobile、`>= 768px` 视为非 mobile 的明确边界（左闭右开），避免抖动 |
| 移动端讲解/沙箱切换时沙箱正在运行/有未保存修改 | 切换 Tab 不清空、不重置沙箱代码与运行状态，仅隐藏/显示对应面板（DOM 保留，非卸载重建） |
| 用户系统主题在页面打开期间发生变化（如 macOS 定时切换深色模式） | 若 `mode='system'`，页面主题应实时联动切换；若 `mode` 为用户手动指定的 `light`/`dark`，则不受系统变化影响 |
| 极窄视口（<375px，如部分老旧设备横屏计算异常） | 仍按 mobile 断点规则渲染，允许内容纵向增长，但不得出现横向滚动 |
| 超宽视口（>1440px，如宽屏显示器） | 桌面端布局居中定宽或按比例扩展，不得出现内容被拉伸到不可读的极端行宽 |
| 代码块/表格内容本身超出容器宽度 | 该组件内部允许局部横向滚动（如 `overflow-x: auto` 的代码块），但不得导致页面级 body 横向滚动 |
| 对比度校验未达标的历史遗留配色（如误用仪表盘配色残留） | 视为缺陷需修复配色本身，不允许通过加粗/加边框等方式掩盖对比度问题 |
