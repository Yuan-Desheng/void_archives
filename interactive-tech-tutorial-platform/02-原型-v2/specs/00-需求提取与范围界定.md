# 需求提取与范围界定

> 来源文件：`01-需求/原始需求.md`（阶段①输入，2026-07-17 范式转向后版本）
> 产品：Interactive Tech Tutorial Platform（互动式技术教程平台，暂定名）
> 说明：本文为原型阶段（阶段②）的需求提炼与范围界定，作为原型与后续设计的唯一真源。

---

## 一、背景概述

本平台是一款**内部自用的互动式技术教程工具**，参照 Vue3 官方互动教程（cn.vuejs.org/tutorial）的产品形态打造。核心价值是「**内容与引擎分离**」——教学内容以 JSON 配置（TutorialPackage）驱动，引擎只负责渲染与交互，从而做到「新增一个 JSON 即可跑通一个新主题、引擎代码零改动」。

首个承载的主题是 React（来自「小满zs」62 篇学习笔记，天然分 8 章：入门 / 工具 / 原理 / 组件 / CSS 方案 / Hooks / Router / Zustand），并预留 Vue、JavaScript、算法等主题扩展能力。产品形态在 2026-07-17 经 Owner 决策 D4 完成范式转向：**忠于 Vue3 范式**，砍掉原提示词中的「高亮定位 / 等待机制 / 聚光灯」（R3/R4/R5），确立「**左讲解 + 右沙箱，靠内容引导、不拦人**」的最终形态。

## 二、建设目标

1. **内容引擎分离**：以配置化 TutorialPackage 驱动教学，内容与渲染引擎解耦。
2. **多主题可扩展（成败判据）**：新增一个 `vue.json` 即可跑通 Vue 主题，引擎代码零改动。
3. **对标 Vue3 的学习体验**：左侧 MDX 讲解、右侧可运行代码沙箱、底部「当前步 / 总步数」步骤导航。
4. **真实可运行的代码沙箱**：Sandpack 多文件编辑、可 Run、预览实时刷新、语言模式随主题切换。
5. **一键看答案（R10）**：每步提供 solution 代码，一键填入沙箱并可运行，可切回自己的代码。
6. **统一克制的视觉（D5）**：shadcn neutral 纯灰阶、Inter 字体、radius .5rem、亮/暗双主题，对比度达 WCAG AA。
7. **响应式**：375 / 768 / 1440 三档无横向滚动、无重叠；移动端讲解与沙箱可切换（不并排挤）。

## 三、用户角色

| 角色 | 说明 |
|---|---|
| 学习者 | 平台主要使用者，阅读讲解、在沙箱中动手写代码、看答案、按步骤推进学习进度。 |
| 内容维护者 | 编写 / 组织 TutorialPackage（章节、步骤、初始代码、答案代码），新增主题 JSON。 |
| Owner（项目负责人） | 内部教具的定义者，把控范式与范围边界（本项目即演示用户本人自用场景）。 |

## 四、纳入范围（inScope）

原型阶段（阶段②）明确要做，且可落地为界面：

- **R1 内容展示**：按章节 / 步骤渲染 MDX 讲解，含代码块语法高亮。
- **R2 步骤引导**：上一步 / 下一步导航，进度指示反映「当前步 / 总步数」（对标 Vue3 `1 / 15`）。
- **R6 代码沙箱壳**：Sandpack 编辑器 + 预览（上下分栏），多文件切换，语言模式随主题切换。
- **R10 给我看答案**：每步 solution 一键填入沙箱、可运行、可切回自己的代码。
- **R8 多主题扩展结构**：主题列表 / 切换，配置化课程包（TutorialPackage）的目录与章节结构呈现。
- **R7 响应式**：三档断点适配，移动端讲解 / 沙箱可切换。
- **R9 外观主题**：shadcn 灰阶、亮 / 暗双主题切换。
- **拟真数据**：以 React 主题的 1–2 章真实笔记内容做原型示例数据。
- **进度记忆（原型级）**：以 localStorage 记录当前步 / 已学进度（原型阶段，不落库）。

## 五、不纳入范围（outOfScope）及理由

| 内容 | 不纳入原因 |
|---|---|
| 引导蒙层 / 聚光灯 / 高亮框 / waitFor 阻塞（R3/R4/R5） | D4 已砍，与 Vue3「不拦人」范式冲突，明令不得再出现 |
| 深色仪表盘配色（#4C6FFF / #22D3EE / #FDBA33） | D5 已改 shadcn 灰阶，视觉方向已废弃 |
| 用户系统 / 登录（Clerk） | 原型阶段不做；MVP 才引入，属后端与鉴权范畴，非界面原型重点 |
| 进度持久化到 PostgreSQL + Drizzle | 原型阶段用 localStorage 即可，DB / ORM 属后端基础设施 |
| Vue / JavaScript / 算法主题的实际内容 | 只保证数据结构可扩展，原型不产出这些内容 |
| Sandpack / WebContainer 真实运行时的底层实现 | 属技术实现与运行时选型，原型用沙箱壳 + 拟真交互表达即可 |
| 62 篇笔记的全量搬运 | 原型只需样本章节，全量内容转写是开发阶段成本项（Q3） |
| 技术选型细节（Next.js 15 / Drizzle / Vercel 部署等） | 属架构与运维，非界面功能 |
| 62 篇笔记如何转 steps、章节顺序谁定（Q3/Q4） | 内容工程问题，不阻塞原型，留待开发阶段确认 |

## 六、功能模块总览

基于 inScope 按业务边界拆分（MECE）：

| key | 模块名 | 职责摘要 |
|---|---|---|
| theme-library | 主题课程库 | 展示可学主题列表与切换，呈现每个 TutorialPackage 的章节结构，承载 R8 多主题可扩展。 |
| learning-workspace | 学习工作台 | 核心页：左 MDX 讲解 + 右代码沙箱 + 底部步骤导航（上一步/下一步、当前步/总步数），承载 R1/R2。 |
| code-sandbox | 代码沙箱 | Sandpack 多文件编辑器、实时预览、Run，以及「给我看答案」solution 填入与切回，承载 R6/R10。 |
| learning-progress | 学习进度 | 进度指示、以 localStorage 记录当前步与已学章节，支持继续学习，承载 R2 的进度侧与原型级记忆。 |
| appearance-theme | 外观与主题 | shadcn 灰阶、亮/暗双主题切换、三档响应式与移动端讲解/沙箱切换，承载 R7/R9。 |

## 七、样式基调判断

系统为内部教具 / 内容驱动的教程工具，视觉上明确要求 shadcn neutral 纯灰阶、克制排版、亮暗双主题，且**明令禁止仪表盘式炫彩配色**。调性偏结构化、克制、工程化的管理/工具类界面，故 styleHint 取 **gov**（政企/工程/管理类的克制风格），而非监测/可视化的 dashboard。
