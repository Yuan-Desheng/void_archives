# 模块功能定义：代码沙箱（code-sandbox）

> 依据：`02-原型-v2/prototype/specs/00-需求提取与范围界定.md` 六「功能模块总览」的 code-sandbox 定义。
> 承载需求：R6 代码沙箱壳、R10 给我看答案。
> 前提（2026-07-17 D4 范式转向已生效）：**本模块不含高亮/聚光灯/waitFor 阻塞**（R3/R4/R5 已砍，相关字段 `selector`/`waitFor` 已从 Schema 删除）。沙箱靠内容引导学习者自己动手、自己判断，不拦人。

## 0. 模块概述

代码沙箱是「学习工作台」页面右侧（或移动端下半屏）的可真实运行代码环境，基于 [Sandpack](https://sandpack.codesandbox.io/) 构建。它对标 Vue3 官方教程实测到的 `vue-repl`（CodeMirror 编辑器 + iframe/srcdoc 预览，`split-pane vertical`），并在此之上落地本项目独有的 **R10「给我看答案」**——每一步骤除了初始代码 `files`，还可声明答案代码 `solution`，学习者卡壳时一键把 `solution` 填入沙箱、运行、看效果，看完还能一键切回自己刚才写的代码。

沙箱与「学习工作台」（步骤导航）解耦：工作台只负责在进入某一步时把该步骤的 `files` 交给沙箱渲染；沙箱本身不关心自己处在教程的第几步、不做进度判断，只响应「初始化文件 / 运行 / 看答案 / 切回我的代码」这几类外部指令与用户交互。

核心能力：
- 多文件工程结构的编辑与切换（如 `App.js` + `styles.css` + `index.js`）
- 真实的浏览器端打包运行（依赖 Sandpack bundler，而非静态代码高亮展示）
- 手动 Run（而非逐字符自动编译，理由见业务规则）与实时预览面板
- 语言模式随主题切换：`TutorialPackage.meta.language` 决定沙箱的 `template`（`react` / `vanilla` / 未来 `vue`），组件本身不写死任何主题分支——这是 R8「新增一个 JSON 即可跑通新主题、引擎零改动」在沙箱侧的落地
- 步骤级初始代码渲染（`files`）
- **给我看答案（R10）**：`solution` 一键填入、可运行、可切回自己的代码
- 375 / 768 / 1440 三档响应式布局，移动端讲解区与沙箱区可切换（非并排）

沙箱**不负责**：步骤导航、进度记录、高亮定位、等待/阻塞判定（这些要么已被 D4 砍掉，要么是「学习工作台」「学习进度」模块的职责）。沙箱只是一体化的编辑-运行-看答案组件。

## 1. 用户故事

- 作为**学习者**，读完左侧一段讲解后，我希望右侧沙箱已经有该步骤对应的示例代码，点 Run 就能看到效果，而不用自己抄代码。
- 作为**学习者**，我希望能自由修改沙箱里的代码（比如改一个 prop 的值），点 Run 后立刻看到预览变化，用来验证我对知识点的理解——过程中平台不会以任何形式打断我或判定我对错。
- 作为**学习者**，遇到某一步卡壳看不懂时，我希望点一下「给我看答案」，该步骤的标准答案代码立刻填入沙箱并自动跑起来，让我直接看到"正确写法长什么样"。
- 作为**学习者**，看完答案后，我希望能一键"切回我的代码"，回到刚才自己写的（哪怕没写完的）版本，而不是被答案覆盖后再也找不回来。
- 作为**学习者**，我希望在手机上也能看代码、跑代码，通过"讲解 / 代码"两个 Tab 分别看清楚，而不是被挤成无法阅读的两栏。
- 作为**学习者（暗色模式用户）**，我希望沙箱编辑器和预览面板的配色跟随平台整体明暗主题切换，不要出现"页面暗色、编辑器刺眼纯白"的割裂体验。
- 作为**内容维护者**，我希望只需要给每个 step 配置好 `files`（初始代码）和 `solution`（答案代码），沙箱组件就能自动渲染与响应，不需要为具体某一步骤额外写渲染逻辑。
- 作为**内容维护者**，我希望新增 Vue / JavaScript 主题时，只需在 `TutorialPackage.meta.language` 声明对应模板，沙箱组件不用为新主题改代码。
- 作为**Owner**，我要求 R8「新增一个 JSON 即可跑通新主题、引擎零改动」是可验收的成败判据，沙箱侧必须证明：组件永远从教程包的 `meta.language` 读取模板，不允许存在任何写死主题分支的代码。

## 2. 功能点清单

| 编号 | 功能点 | 说明 |
|---|---|---|
| F1 | 多文件切换 | 沙箱内以文件 Tab 形式展示当前步骤涉及的多个文件，学习者可点击切换编辑区显示的文件 |
| F2 | 代码编辑 | 基于 CodeMirror（Sandpack 内置）的可编辑代码区，支持语法高亮、基础自动缩进 |
| F3 | Run 运行 | 点击 Run 按钮触发 Sandpack bundler 重新打包并刷新预览面板 |
| F4 | 实时预览 | 预览面板（iframe 沙箱）展示 Run 后的渲染结果 |
| F5 | 语言模式随主题切换 | 沙箱的 `template`（如 `react` / `vanilla` / `vue`）由当前 `TutorialPackage.meta.language` 决定，组件不写死 |
| F6 | 步骤初始代码渲染 | 进入某一步骤时，用该步骤 `files` 字段的内容渲染沙箱的文件集合与内容 |
| F7 | 给我看答案（R10） | 点击「给我看答案」按钮，把当前步骤 `solution` 声明的文件内容整体填入沙箱、自动触发一次 Run |
| F8 | 切回我的代码（R10） | 处于"看答案"态时提供「切回我的代码」按钮，还原为点击看答案前的用户编辑快照，并自动触发一次 Run |
| F9 | 移动端布局降级 | ≤768px 时讲解区与沙箱区从左右并排改为上下 Tab 切换，避免横向挤压导致代码不可读 |
| F10 | 明暗主题联动 | 沙箱编辑器与预览面板的配色跟随平台主题（`light` / `dark`）实时切换 |
| F11 | 运行状态反馈 | Run 过程中展示 loading 态；编译报错时在预览区展示 Sandpack 原生错误堆栈（不吞错） |
| F12 | 依赖声明 | 教程包可声明 `dependencies`（npm 包名+版本），沙箱据此加载依赖，供示例代码 import 使用 |

## 3. 业务规则

1. **Run 触发方式**：默认采用"手动 Run 按钮触发"，不做逐字符自动编译刷新（自动编译在低性能设备上易造成卡顿，且与"学习者自己判断是否完成"的 Vue3 范式更契合——不替学习者做决定）。
2. **步骤切换时的文件渲染**：进入新步骤时，用该步骤的 `files` **完整替换**当前沙箱文件集合（不做跨步骤的增量合并）。理由：62 篇笔记彼此独立、各自成篇，每一步应视为一次独立的示例代码，避免"上一步残留代码"污染当前步骤的示例。
3. **看答案的填入范围**：点击「给我看答案」时，先把学习者当前的编辑内容（`currentFiles`）快照保存为 `ownCodeSnapshot`，再用该步骤的 `solution` **完整替换**沙箱文件集合，并自动 Run 一次；此时沙箱进入 `isShowingAnswer = true` 态。
4. **切回我的代码**：点击「切回我的代码」时，用 `ownCodeSnapshot` 还原沙箱文件集合并自动 Run 一次，`isShowingAnswer` 复位为 `false`，`ownCodeSnapshot` 清空。若学习者在"看答案"态下又编辑了答案代码，这些编辑不会被保留——直接丢弃、还原为看答案前的快照（不做二次确认弹窗，遵循简洁守则）。
5. **步骤切换时自动退出看答案态**：无论当前是否处于 `isShowingAnswer`，切换到上一步/下一步时都强制复位为"我的代码"模式（用新步骤的 `files` 渲染），不把上一步骤的看答案状态带入下一步骤，避免状态叠加错乱。
6. **`solution` 缺失时的降级**：若当前步骤没有声明 `solution` 字段，「给我看答案」按钮禁用（置灰，不可点），不弹错误提示——这是内容维护者尚未补充该步骤答案的正常情况，不是异常。
7. **语言模式与依赖锁定**：`meta.language` 与 `dependencies` 由 `TutorialPackage` 决定，运行时沙箱组件不提供"切换成其他语言模板"的入口，保证与教程内容严格对应。
8. **移动端交互**：≤768px 断点下，默认展示"讲解"Tab，学习者需手动切到"代码"Tab 才能看到沙箱；沙箱内多文件 Tab 与讲解内容的 Tab 是两层独立的 Tab 结构，不做嵌套折叠。
9. **主题联动优先级**：沙箱主题跟随平台明暗模式设置，不提供沙箱独立的主题开关（非核心功能，遵循简洁守则不加）。
10. **文件路径缺失降级**：若 `files` 或 `solution` 引用的文件路径此前不存在，沙箱自动新建该文件并追加到文件 Tab 列表，不报错阻断。

| 页面 key | 页面名称 | 职责一句话 |
|---|---|---|
| sandbox-panel | 沙箱主面板（桌面右栏） | 承载多文件 Tab + 编辑区 + Run 按钮 + 预览区 + 「给我看答案 / 切回我的代码」的一体化沙箱视图，桌面 ≥768px 下与讲解区左右并排 |
| sandbox-mobile-tab | 移动端沙箱 Tab 视图 | ≤768px 下作为讲解/代码两个 Tab 之一，全宽承载沙箱主面板的精简版布局 |
| sandbox-error-state | 沙箱异常态视图 | 展示编译报错堆栈、依赖加载失败提示等异常反馈，嵌入在主面板预览区内而非独立跳转页面 |

## 5. 关键数据字段

### TutorialPackage（教程包，沙箱相关字段）
- `id`: string（不枚举限定，任意主题标识，如 `"react"` / `"vue"`）
- `meta.language`: string，沙箱语言模式（对应 Sandpack `template`，如 `react` / `vanilla` / `vue`）
- `meta.defaultCode`: string，默认代码模板（教程包级兜底）
- `dependencies`（可选）: `{ [pkgName: string]: string }`，教程包级 npm 依赖声明

### Step（步骤，沙箱相关字段）
- `files`（可选）: `{ [filePath: string]: string }`，本步骤的初始代码文件集合（Sandpack 多文件）
- `solution`（可选）: `{ [filePath: string]: string }`，本步骤的答案代码文件集合（R10）

### 沙箱运行时状态（前端内部，不持久化）
- `currentFiles`: `{ [filePath: string]: string }`，当前编辑区实际内容
- `activeFile`: string，当前选中的文件 Tab
- `isRunning`: boolean
- `hasError`: boolean / `errorMessage`: string
- `theme`: `"light" | "dark"`
- `isShowingAnswer`: boolean，是否处于"看答案"态
- `ownCodeSnapshot`: `{ [filePath: string]: string } | null`，点击看答案前的用户编辑快照，供"切回我的代码"还原
- `lastRunTimestamp`: number

## 6. 异常 / 边界

| 场景 | 处理方式 |
|---|---|
| 当前步骤未声明 `solution` | 「给我看答案」按钮禁用（置灰），不弹错误提示，属正常的内容未补充状态 |
| 当前步骤未声明 `files` | 沙箱渲染教程包级 `meta.defaultCode` 作为兜底初始内容，不留空白编辑区 |
| `files` / `solution` 中引用的文件路径此前不存在 | 自动新建文件并加入 Tab 列表，不报错、不阻断 |
| `files` / `solution` 字段格式非法（如非对象、值非字符串） | 保留上一步骤沙箱状态不变，控制台记录告警，不弹出用户可见错误 |
| Sandpack bundler 编译失败（学习者或答案代码有语法/运行时错误） | 预览区展示 Sandpack 原生错误堆栈，不吞错、不静默失败；Run 按钮保持可再次点击 |
| 依赖包加载失败（网络问题 / 包名不存在） | 预览区展示依赖加载失败提示，编辑区仍可正常编辑，不影响其他已加载文件的可读性 |
| 断网/离线状态下进入沙箱 | Sandpack bundler 依赖网络加载依赖包，离线时预览区展示"网络异常，无法加载运行环境"提示，编辑区仍可用于查看/编辑代码文本 |
| 学习者在"看答案"态下继续编辑答案代码，之后点「切回我的代码」 | 对答案代码的编辑不保留，直接丢弃，还原为看答案前的 `ownCodeSnapshot`；不做二次确认弹窗（遵循简洁守则） |
| 学习者未点 Run 就切换到上一步/下一步 | 未运行的编辑内容随步骤切换丢弃，不做自动保存草稿（非核心功能，暂不实现） |
| 移动端窄屏（375px）多文件 Tab 过多导致横向溢出 | 文件 Tab 区域单独可横向滚动（`overflow-x: auto`），不引发整页横向滚动 |
| 学习者快速连续点击 Run | 对 Run 做防抖/禁用态处理，进行中不重复触发新的打包任务 |
| 极端超长代码文件（如粘贴大段无关代码） | 不做字数上限硬限制（内部工具场景无需防滥用），编辑区渲染性能依赖 CodeMirror 自身，不额外优化 |
