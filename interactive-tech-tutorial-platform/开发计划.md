## 一、这个项目到底在做什么（一句话）

**做一个渲染器，不是做一个 React 教程。** React 只是塞进去的第一包内容（62 篇现成笔记）。
判断成败的标准只有一条：**加一个 `vue.json` 就能跑出 Vue 教程，不改任何代码**（对应 R8）。
如果最后做出来的是「一个好看的 React 教程网站，但换主题要改代码」——这个项目就失败了。

> **D4 之后，这句话的分量更重了**：砍掉引导引擎后，本项目的技术含量几乎只剩两件事——
> ① Sandpack 集成、② **配置驱动的多主题渲染（R8）**。
> ①是接第三方库，②才是真正属于这个项目的东西。**R8 现在是唯一的核心难点，不是之一。**

## 二、已定决策（2026-07-17，详见 `99-会议与决策/`）

| 决策 | 结论 | 对计划的影响 |
|---|---|---|
| **D1 沙箱** | Sandpack 真实运行 | 引入 `@codesandbox/sandpack-react`；移动端要做体积/布局降级 |
| ~~**D2 引导对象**~~ | ~~仅平台自身 UI（host）~~ | ❌ **已被 D4 作废**：无聚光灯 → 无 selector → 该议题自然消失 |
| **D3 后端** | Clerk + Postgres + Drizzle 全套进 MVP | 不是纯静态站；工期 +1 周；Vercel 需配 Neon/Supabase + Clerk 环境变量 |
| **D4 教学范式** 🔴 | **Vue3 式自主学习，砍掉 R3/R4/R5** | **引导引擎不再存在**；产品 = 左讲解 + 右沙箱；工期大幅缩短 |
| **D5 视觉** 🔴 | **shadcn neutral 灰阶**（纯灰阶零彩色 + Inter + radius .5rem） | 原型 18 页配色重做；亮暗双主题由 shadcn token 天然提供 |

> [!danger] D4 是本项目至今最大的一次转向
> **起因**：Playwright 实地扒取 Vue3 教程 DOM，发现它 **`highlight`/`spotlight`/`mask` 零命中**——
> 根本没有聚光灯。「类似 Vue3 教程」与「R3/R4/R5」是**方向相反的两个范式**，不可兼得。
>
> **代价**：v1 原型中已实测可跑的 446 行引导引擎作废，`guided-tour` 5 页报废，18 页配色重做。
> **收益**：在写第一行 Next.js 之前发现矛盾。代价 = 重跑一次原型（~30min 机器时间）；
> 若在阶段④后发现 = 推翻引擎 + 返工 Schema + 重做全部页面。**这就是原型驱动的意义。**

## 三、最大风险：内容，不是代码（必须先看这条）

> [!danger] 全项目最大成本项是「62 篇笔记 → tutorial steps」，不是写引擎
> 引擎大约 2 周能出。**内容转换可能吃掉 3–4 周**，而且是纯体力活、AI 不一定省力。
> 盘点结果：canvas 62 节点、**0 条 edge**（全孤立）→ 章节顺序无法推导，得人工定。
> Router 24 篇 + Hooks 17 篇占了 66%，这两块是重灾区。

**Q3（未决）**：62 篇怎么转？三条路，成本差一个量级：

| 路径 | 做法 | 成本 | 产出质量 |
|---|---|---|---|
| **A · 直接 MDX 渲染** | 笔记原文当 MDX 直接渲染，不拆 steps，只在章节级做导航 | ~3 天 | 就是个笔记网站，R3/R4/R5 全废 |
| **B · AI 批量转 + 人工校**（建议） | AI 把每篇拆成 steps JSON，人工校验 | ~1.5 周 | 中等，需抽检臆造 |
| **C · 人工逐篇改写** | 一篇篇手写 steps + 配套 defaultCode | 3–4 周 | 最好 |

**建议**：MVP 只做 **8–10 篇样板**（入门3 + Hooks 核心5），用路径 C 打磨到最好，验证引擎；剩下 52 篇留到 v2 用路径 B 批量补。
**理由**：先证明「引擎 + 一包好内容」成立，再规模化。62 篇全量搬运是典型的「先把体力活干完再发现方向错了」。

**Q4（未决）**：章节顺序谁定 → 建议 Owner 直接拍一版线性顺序（入门 → 组件 → Hooks → CSS → Router → Zustand → 工具 → 原理），不必完美。

### 阶段③ 设计
- 工具：`工作流-设计`（proto-to-design）→ `03-设计/`
- 必须回答的技术点：
  - **引擎抽象**：TutorialPackage → 运行时状态机的映射；selector 失效/元素不存在时的降级（DoD 要求「不崩」）
  - **聚光灯实现**：SVG mask vs `box-shadow: 0 0 0 9999px` vs backdrop-filter — 三选一，要有理由
  - **Sandpack 集成**：bundle 体积、SSR 边界、移动端降级策略
  - **数据库 ER**：user / progress / tutorial_package —— 进度粒度到 step 还是 chapter？
  - **接口契约**：进度读写 API
- **建议先做 spike**（规范 §五③ 允许并行）：聚光灯 + Sandpack 在移动端共存，这是最可能翻车的点，先写个 50 行 demo 验
- DoD：架构图/数据流/ER/接口契约齐全 + 选型有理由 + 安全运维非功能性有方案

### 阶段④ 开发（D4 后重排）
- 工具：`工作流-开发`（design-to-code）
- 拆成可独立上线的小步：
  1. 脚手架：Next.js 15 + TS + Tailwind v4 + shadcn/ui（**默认 neutral 主题，零改色**）
  2. **TutorialPackage 加载与校验**（无 UI，纯逻辑 + 单测）：JSON → 类型安全的运行时对象；**这是 R8 的落点**
  3. 内容层：MDX 渲染 + 章节/步骤导航 + `当前步/总步数`
  4. Sandpack 接入：多文件、Run、实时预览、language 随主题切
  5. R10「给我看答案」：solution 填入沙箱 + 可切回
  6. 后端：Drizzle schema + 幂等迁移 + 进度 API + Clerk
  7. 响应式打磨（移动端讲解/沙箱 Tab 切换）
- ~~引擎核心 / 引导 UI 层~~ —— **D4 已砍，不存在**
- 前端复用原型 v2（规范 §五④）
- **R8 验收用例（必做）**：写完 React 主题后，**新增一个 `vue.json` 跑通 Vue 教程，且 diff 显示零引擎代码改动**。做不到 = R8 未达成。
- 人工必检：TutorialPackage 校验的边界（字段缺失 / chapters 为空 / solution 不存在）；公开库零明文凭据
