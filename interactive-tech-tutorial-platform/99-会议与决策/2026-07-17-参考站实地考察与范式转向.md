## 一、实地考察结论

### 1.1 Vue3 官方教程（cn.vuejs.org/tutorial）—— **没有聚光灯**

实测 DOM：

| 探测项 | 实际结果 |
|---|---|
| `[class*=highlight]` / `[class*=spotlight]` / `[class*=mask]` | **0 命中** |
| 布局 | 左 `.vt-doc`（讲解）+ 右 `vue-repl`（`split-pane vertical`） |
| 编辑器 | **CodeMirror**（非 Monaco） |
| 预览 | `iframe` + `srcdoc` + `sandbox="allow-forms allow-modals allow-pointer-lock allow-popups allow-same-origin allow-scripts allow-top-navigation-by-user-activation"` |
| 步骤 | 15 步（`1 / 15`），「下一步」按钮推进，**无阻塞、无等待检测** |
| 引导方式 | **靠内容质量**，学习者自行判断是否完成 |
| 特色 | 每步提供「给我看答案」（solution 代码） |

**结论**：Vue3 教程的范式是「**左边讲、右边写、自己跑**」，全程不拦人。

### 1.2 aura.build/docs —— shadcn 默认灰阶

| 探测项 | 实际结果 |
|---|---|
| 设计 token | 标准 shadcn/ui `neutral` 主题（HSL 格式） |
| 亮色 | `--background: 0 0% 100%` / `--foreground: 0 0% 9%` / `--primary: 0 0% 9%` |
| 暗色 | `--background: 0 0% 9%` / `--foreground: 0 0% 98%` / `--muted-foreground: 0 0% 65%` |
| 色相 | **纯灰阶，零彩色**（所有 token 饱和度为 0%） |
| 字体 / 圆角 | Inter / `--radius: .5rem` |
| 布局 | 含 `--sidebar-*` token → 文档站左侧栏布局 |

> ⚠️ **如实记录**：`/docs` 路由**可见文本仅 12 字符**，SPA 内容未渲染。
> 只取到设计 token，**未能看到该文档站的实际视觉与交互**。此参考链接的有效性需 Owner 复核。

## 二、发现的核心矛盾（本次考察最大价值）

> [!danger] 原始需求自相矛盾
> 提示词同时要求：
> - **「类似 Vue3 官方互动教程」**（定位）
> - **R3 高亮定位 + R4 等待机制 + R5 聚光灯**（功能）
>
> 但实测证明 **Vue3 教程完全没有 R3/R4/R5**。
> R3/R4/R5 属于**产品 onboarding tour**（Intro.js / Shepherd 一类）的范式，
> 与 Vue3「自主学习」范式**方向相反**。二者不可兼得。

## 三、决策（Owner 拍板）

### D4 · 教学范式 = Vue3 式自主学习，**砍掉 R3/R4/R5**
**选项**：Vue3 式（砍 R3/R4/R5）/ 聚光灯式（放弃对标 Vue3）/ 两层（首访导览 + Vue3 式正文）
**决策**：**Vue3 式，砍掉 R3/R4/R5**
**理由**：忠于「类似 Vue3 官方互动教程」这一核心定位；符合「删 > 加」原则。

**后果（代价必须记录）**：
- ❌ 原型中 446 行引导引擎状态机**作废**（该引擎已于同日实测证明可真实运行——聚光灯定位精确、waitFor 阻塞有效、自动推进正常）
- ❌ `guided-tour` 模块 5 个页面基本报废
- ✅ **D2 自动失效**：无聚光灯 → 无 `selector` → 「scope: host | sandbox」议题**自然消失**，无需再决策
- ✅ 工期大幅缩短：引擎从「核心难点」降级为「不存在」
- ✅ 产品定位澄清：**内容与沙箱是主角，引擎不是**

### D5 · 视觉风格 = shadcn 灰阶（向 aura.build 靠）
**选项**：shadcn 灰阶 / 保留深色仪表盘 / 灰阶打底+金黄聚光灯
**决策**：**shadcn 灰阶**
**理由**：与已定选型 shadcn/ui 天然一致，默认主题直接可用，零额外设计成本。
**后果**：
- ❌ 原型 18 页配色全部重做（深色仪表盘 #4C6FFF/#22D3EE/#FDBA33 → shadcn neutral 灰阶）
- ✅ 亮暗双主题由 shadcn token 天然提供
- 注：D4 砍掉聚光灯后，金黄强调色（#FDBA33）已失去语义载体，一并移除

## 四、Schema 变更（D4 的直接后果）

原 Schema 中 `selector` / `waitFor` 是为 R3/R4/R5 服务的，**现已成死字段，必须删除**。
同时实测发现 Vue3 每步提供「给我看答案」，原 Schema **缺 solution 字段**，需补。

```typescript
interface TutorialPackage {
  id: string;                    // 改：不再枚举限定，否则违背 R8「新增主题无需改引擎」
  meta: {
    title: string;
    description: string;
    language: string;            // 沙箱语言模式
    defaultCode: string;         // 默认代码模板
  };
  chapters: {
    id: string;
    title: string;
    steps: {
      id: string;
      title: string;
      description: string;       // MDX 讲解
      files?: Record<string, string>;     // 新增：该步初始代码（Sandpack 多文件）
      solution?: Record<string, string>;  // 新增：答案代码（对标 Vue3「给我看答案」）
      // selector  —— 删除（R3 已砍）
      // waitFor   —— 删除（R4 已砍）
      // codeSnippet —— 由 files 取代（Sandpack 需多文件，单字符串不够）
    }[];
  }[];
}
```

> `id: 'react' | 'vue' | 'javascript'` 改为 `string`：
> 原枚举写死三个主题，与 **R8「新增 JSON 即可跑通新主题、无需改引擎代码」直接冲突**——
> 加第四个主题就得改类型定义 = 改引擎。这是原 Schema 的硬伤。
