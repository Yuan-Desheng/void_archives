## D6 决策

**一期 = 纯前端 Next.js + Sandpack + Vercel 静态部署，零后端。**

| 项 | D3（原） | D6（现·一期） |
|---|---|---|
| 认证 Clerk | ✅ 进 MVP | ❌ **砍**（一期不做多用户） |
| PostgreSQL + Drizzle | ✅ 进 MVP | ❌ **砍** |
| Route Handler（/api/progress、/api/webhooks/clerk） | ✅ | ❌ **砍** |
| users 表 / learning_progress 表 | ✅ | ❌ **砍**（两表都不建） |
| 学习进度 | 跨端同步（Postgres） | **localStorage**（本机） |
| 内容读取 | RSC 读 JSON | ✅ **保留**（RSC/SSG 读 JSON，纯静态） |
| 跨端同步 | 一期做 | **降级二期** |

## 理由

- 一期就 Owner 自己用，**不做多用户** → 身份系统（Clerk）失去意义。
- 不做多用户 → 进度无需按用户隔离 → localStorage 足够。
- 符合「删 > 加」：砍掉后端 = 砍掉 DB 运维、Clerk 配置、Webhook 同步、鉴权中间件一整条链路。

## 代价（如实记录）

- ❌ **换设备 / 清浏览器缓存 → 进度丢**。这是 D3 当初要「跨端同步」的诉求，现降级二期。
- Owner 已知情接受：一期「先自己用」阶段，进度丢可接受。

## 对既有设计文档的处理

**不重跑设计工作流**（这是范围收窄，架构骨架仍有效，重跑是浪费）。处理方式：
- 设计文档 `03-设计/specs/` 的 **database / api / security / ops 中涉及 Postgres/Clerk/Route Handler 的部分 = 二期参考**，一期不实现。
- 架构骨架（五层分层、R8 内容-引擎分离、Sandpack 客户端、内容 JSON 单一真源、shadcn 灰阶）**一期照做**。
- 一期实现范围以 `03-设计/一期实现范围.md` 为唯一真源，开发阶段照此。

## 一期后端相关维度的去留

| 设计维度 | 一期 |
|---|---|
| architecture | ✅ 照做，但删去 L3 接口层 + L4 持久层（localStorage 取代） |
| dataflow | 保留内容渲染流 + 看答案流；**进度流改 localStorage 单机版** |
| database | ❌ **一期不做**（无 DB）。二期参考 |
| api | ❌ **一期不做**（无 Route Handler）。内容走 RSC/SSG，无 HTTP 端点 |
| security | 大幅简化：无鉴权、无 DB、无 PII 存储 → 一期几乎只剩「公开仓零明文 + 依赖安全」 |
| ops | 简化为「Vercel 静态部署 + 环境变量（无 DB/Clerk 密钥）」 |
| nfr | ✅ 照做（R8 首要目标不变） |
| integration | 只剩 Sandpack 依赖 CDN；砍 Clerk/Postgres 集成 |
