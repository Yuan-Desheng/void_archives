# 系统设计总览

> 阶段③设计 · 唯一真源汇总。综合原型 specs（`02-原型-v2/specs/`）、原型清单（`manifest.proto.json`）与原始需求（`01-需求/`、`99-会议与决策/`）互相印证。
> 产品：**互动式技术教程平台（Interactive Tech Tutorial Platform）**——内容与引擎分离、以 JSON 配置驱动的「左讲解 + 右可运行沙箱」自主学习工具。
> 范式基线：2026-07-17 Owner 决策 D4/D5 之后版本——忠于 Vue3 官方互动教程，**左讲解 + 右沙箱、靠内容引导、全程不拦人**；砍掉 R3/R4/R5（高亮/等待/聚光灯）。

---

## 一、系统概述与设计目标

本平台是一款**内部自用**的互动式技术教程工具，对标 Vue3 官方互动教程（cn.vuejs.org/tutorial）的产品形态。核心价值是「**内容与引擎分离**」：教学内容以 JSON 配置（`TutorialPackage`）驱动，引擎只负责渲染与交互，做到「**新增一个 `vue.json` 即可跑通一个新主题、引擎代码零改动**」——这是本项目的成败判据（R8）。

首个承载主题为 React（来自「小满zs」62 篇笔记，天然分 8 章；MVP 先精做 9 篇样板验证引擎），并预留 Vue / JavaScript / 算法主题扩展。

**设计目标**：
1. **内容引擎解耦**：所有渲染由 `TutorialPackage` 配置数组/树遍历驱动，代码中不得出现针对具体主题（React/Vue）的 if/switch 分支。
2. **对标 Vue3 学习体验**：左 MDX 讲解 + 右可运行 Sandpack 沙箱 + 底部「当前步 / 总步数」步骤导航，步骤永远可点、不阻塞。
3. **真实可运行沙箱**：Sandpack 多文件编辑、手动 Run、实时预览、语言模式随主题切换；每步「给我看答案」（R10）一键填入 solution、可运行、可切回自己的代码。
4. **统一克制视觉**：shadcn neutral 纯灰阶 + Inter + radius .5rem + 亮/暗双主题，对比度达 WCAG AA。
5. **响应式**：375 / 768 / 1440 三档无横向滚动、无重叠；移动端讲解/沙箱可切换。
6. **进度记忆**：原型阶段用 localStorage；MVP（Q5 决策）落 PostgreSQL + Clerk 用户系统，进度跨端同步。

## 二、架构形态建议（architectureStyleHint）

**分层单体（Next.js 全栈单体）**。Next.js 15 App Router 一体承载前端渲染引擎与后端 Route Handler，Drizzle + PostgreSQL 存进度，Clerk 管鉴权，Vercel 部署——业务规模是内部自用小工具，微服务/中台纯属过度设计；「内容与引擎分离」是**模块边界与配置驱动的架构原则**，不是部署层面的服务拆分。

## 三、技术约束（techConstraints）

> ⚠️ 与典型政企/信创项目相反：本项目**不是**内网/专网/等保场景，如实记录以免下游误套政企约束。

- **部署于 Vercel 公有云**：非内网/非专网、无信创国产化要求、无等保定级、数据不涉及「不出域」限制。
- **内部自用工具**：无对外合同、无交付评审对象（按流程规范 §3.4 走简化流程），Owner 即定义者。
- **内容-引擎分离为硬约束（R8 成败判据）**：引擎代码零主题分支，主题包以 JSON 配置形式独立于引擎；架构须在阶段③给出「新增 JSON 零改动」的架构级证明。
- **数据结构以现行 Schema 为唯一真源**：`TutorialPackage`（已删死字段 `selector`/`waitFor`，`codeSnippet` 由多文件 `files` 取代，新增 `solution`）；数据库/接口设计须按 9 篇样板的真实 step 形态、而非 62 篇的想象来设计。
- **范式禁令（D4）**：不得出现引导蒙层/聚光灯/高亮框/waitFor 阻塞，任何设计稿或代码复现视为范式违规。
- **视觉禁令（D5）**：禁止仪表盘式高饱和撞色（`#4C6FFF/#22D3EE/#FDBA33` 已废弃），统一 shadcn neutral 灰阶。
- **既定技术选型**：Next.js 15 + React 19 + TypeScript + Tailwind v4 + shadcn/ui + Drizzle + PostgreSQL + Clerk + Vercel（用户已定）。
- **沙箱运行时依赖**：Sandpack 浏览器端 bundler（`@codesandbox/sandpack-react`），运行发生在客户端。
- **进度存储分期**：原型 localStorage → MVP PostgreSQL，需按 Q5 决策一次做全套后端，不留半吊子。

## 四、非功能性驱动因素（nfrDrivers）

- **用户规模：极小**——内部自用教具，单人/小团队使用，并发峰值个位数~两位数，无高并发设计需求。
- **数据量级：小**——内容侧 React 62 篇笔记 / 8 章（MVP 先 9 篇样板）；进度侧每用户每主题一条断点记录 + 已完成步骤集合，DB 数据量可忽略。
- **实时性：无硬要求**——非监测/调度类，无实时推送需求；沙箱预览的「实时」是客户端 Sandpack 编译，服务端无实时压力。
- **计算分布**：代码打包运行是**客户端计算**（Sandpack bundler 在浏览器内），服务端仅承担页面渲染 + 进度读写，负载轻。
- **可用性：无 SLA 硬约束**——内部工具，Vercel 默认可用性即可满足；进度丢失（清缓存/换设备）在原型级属可接受代价。
- **可扩展性是首要非功能目标**：真正的「非功能压力」不在性能而在**主题可扩展**——新增主题 JSON 零改动引擎（R8）是架构必须扛住的核心指标。

## 五、候选领域实体（entities）

> 数据库设计起点。内容侧（TutorialPackage/Chapter/Step）多以 JSON 配置承载，可存为配置文件或内容表；进度侧（User/Progress）是 MVP 落库的核心。

| 实体 | 含义与关键属性 |
|---|---|
| 教程主题包 TutorialPackage | 一个可学主题（react/vue/...）；`id`(string，不枚举限定) / `meta.title` / `description` / `language`(沙箱语言模式) / `defaultCode`；聚合 chapters。R8 核心配置单元。 |
| 章节 Chapter | 主题包下的章节；`id` / 所属 packageId / `title` / 顺序 order；聚合 steps。 |
| 步骤 Step | 章节下最小学习单元；`id` / 所属 chapterId / `title` / `description`(MDX 讲解) / 顺序 order；含初始代码与答案。 |
| 步骤初始代码 StepFiles | Step 的 Sandpack 多文件初始代码 `files: Record<path,content>`（如 App.js + styles.css + index.js）。 |
| 步骤答案 StepSolution | Step 的答案代码 `solution: Record<path,content>`，承载 R10「给我看答案」。 |
| 用户 User | MVP 引入（Q5），由 Clerk 管理身份；进度归属主体。 |
| 学习进度 LearningProgress | 单用户单主题的记忆；`userId` / `packageId` / 当前 `chapterId`+`stepId`(断点) / 已完成步骤集合 completedSteps / 完成度。原型级存 localStorage，MVP 落 PostgreSQL 并跨端同步。 |

## 六、外部集成清单（integrationPoints）

- **Sandpack（@codesandbox/sandpack-react）**：浏览器端多文件代码打包与运行时（CodeMirror 编辑器 + iframe/srcdoc 预览），沙箱核心依赖。
- **Clerk**：用户认证与身份管理（MVP 引入，Q5 决策）。
- **PostgreSQL + Drizzle ORM**：进度等数据持久化（MVP）。
- **Vercel**：部署与托管平台。
- **内容源（小满zs-react 笔记 / MDX）**：62 篇 React 笔记转 steps 的内容工程输入（属内容侧，非运行时集成）。

## 七、功能模块总览（modules）

沿用原型 specs 的 5 个模块（MECE 拆分，`handoff.styleKey = gov`）：

| key | 模块名 | 职责摘要 | 承载需求 |
|---|---|---|---|
| theme-library | 主题课程库 | 入口与目录层：主题包列表/切换 + 章节/步骤大纲树，纯配置驱动（R8 验收核心）；步骤无锁定、任意可跳转。 | R8 |
| learning-workspace | 学习工作台 | 唯一核心学习页：三段式壳（顶部信息条 + 左 MDX 讲解 + 右沙箱容器 + 底部步骤导航），编排其余模块，不重复实现其逻辑。 | R1、R2 |
| code-sandbox | 代码沙箱 | Sandpack 多文件编辑/手动 Run/实时预览、语言模式随主题切换、「给我看答案」solution 填入与切回。 | R6、R10 |
| learning-progress | 学习进度 | 进度指示可视化（当前步/总步数、进度条、目录打勾、完成度%）+ 原型级 localStorage 记忆、继续学习/从头开始；不做 waitFor 门禁。 | R2 进度侧 |
| appearance-theme | 外观与主题 | 贯穿全站的视觉/布局基础设施：shadcn 灰阶 design tokens、亮/暗切换+跟随系统+记忆、三档响应式与移动端讲解/沙箱切换 Tab。 | R7、R9 |

## 八、视觉风格建议（styleHint）

**gov**。系统为内部教具/内容驱动的工具，明确要求 shadcn neutral 纯灰阶、克制排版、亮暗双主题，且明令禁止仪表盘式炫彩配色。调性偏结构化、克制、工程化的管理/工具类界面，取 gov 而非 dashboard（与 manifest `handoff.styleKey` 一致）。
