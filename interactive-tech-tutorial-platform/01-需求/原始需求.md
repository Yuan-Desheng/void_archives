## 一、项目定位

- 名称：Interactive Tech Tutorial Platform（暂定）
- 参照物：Vue3 官方互动教程 <https://cn.vuejs.org/tutorial>
- 核心价值：**内容与引擎分离**，通过配置化方式驱动交互式教学
- 目标：既能承载既有 React 学习笔记（见「四、内容源」）与 React 官方文档 <https://react.dev/learn>，又能作为通用教具支持多主题（Vue / JavaScript / 算法）扩展
- 样式与交互参考：<https://system-builder.aura.build/docs> 、<https://cn.vuejs.org/tutorial>

## 二、技术选型（用户已定，参考 CodeWithAntonio 频道实践）

| 维度 | 选型 |

## 三、核心功能需求（客户原话逐条，编号供追溯）

| 编号 | 需求 | 原话要点 |

## 五、数据结构设计（用户给的参考 Schema）

```typescript
interface TutorialPackage {
  id: 'react' | 'vue' | 'javascript';
  meta: {
    title: string;
    description: string;
    language: string;      // 代码编辑器语言模式
    defaultCode: string;   // 默认代码模板
  };
  chapters: {
    id: string;
    title: string;
    steps: {
      id: string;
      selector: string;     // CSS 选择器，定位高亮元素
      title: string;
      description: string;  // 支持 Markdown
      waitFor: 'click' | 'input' | 'timer' | 'none';
      codeSnippet?: string; // 当前步骤展示的代码
    }[];
  }[];
}
```

## 六、范围边界

### 原型阶段（阶段②）要做
- R1 内容展示、R2 步骤引导、R3 高亮定位、R4 等待机制、R5 聚光灯、R6 代码沙箱壳、R7 响应式、R9 样式主题
- 以 React 主题的 1–2 章真实笔记内容做拟真数据

### 明确不做（待用户确认，非客户原话，属推断）
- ❌ 用户系统 / 登录（Clerk 标注为「可选、未来」）
- ❌ 进度持久化到 PostgreSQL（原型阶段用 localStorage 即可）
- ❌ Vue / JavaScript / 算法主题的实际内容（只保证数据结构可扩展，不产出内容）
- ❌ 代码沙箱的真实执行（是否需要 WebContainer/Sandpack 级别的运行时未明确，见「七、待确认」）
- ❌ 62 篇笔记的全量搬运（原型只需样本章节）

## 七、已决策项（2026-07-17 Owner 拍板，见 `99-会议与决策/`）

| # | 问题 | 决策 | 后果 |
|---|---|---|---|
| Q1 | 代码沙箱深度 | **Sandpack 真实运行** | 学习者代码可实时预览；引入 `@codesandbox/sandpack-react` |
| Q2 | R3/R4/R5 的 selector 指向谁 | **仅平台自身 UI（host）** | 引擎用 `document.querySelector` 即可，**不做 iframe 探针 / postMessage**；Schema 不加 scope 字段 |
| Q5 | Clerk + PostgreSQL 是否进 MVP | **进，按原选型做全套** | MVP 含后端：Next.js Route Handler + Drizzle + Postgres + Clerk；进度跨端同步 |

> Q2 的语义边界（重要）：教程引导的是「**怎么用这个平台**」——点 Run、切文件、看预览区；
> 「**怎么写 React**」靠 MDX 讲解 + Sandpack 里自己动手。聚光灯**不追**学习者写出来的元素。

## 八、验收口径（初稿）

- R1：能渲染 MDX 笔记，含代码块高亮
- R2：上一步/下一步可点，进度指示反映 当前步/总步数
- R3：给定 selector 能准确高亮目标元素，元素不存在时不崩
- R4：waitFor='click'/'input' 时教程阻塞，用户完成动作后自动进入下一步
- R5：聚光灯遮罩模糊背景，目标元素清晰可见，窗口 resize 后位置正确
- R6：编辑器可输入，language 模式可按主题切换
- R7：375px / 768px / 1440px 三档布局无横向滚动、无重叠
- R8：新增一个 TutorialPackage JSON 即可跑通新主题，无需改引擎代码
