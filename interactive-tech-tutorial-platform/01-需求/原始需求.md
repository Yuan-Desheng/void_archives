## 一、项目定位

- 名称：Interactive Tech Tutorial Platform（暂定）
- 参照物：Vue3 官方互动教程 <https://cn.vuejs.org/tutorial>

**左讲解 + 右沙箱，靠内容引导，不拦人。**

## 五、数据结构设计

### 5.1 原提示词 Schema（已失效，存档备查）

```typescript
interface TutorialPackage {
  id: 'react' | 'vue' | 'javascript';
  meta: { title; description; language; defaultCode };
  chapters: { id; title; steps: {
    id; selector; title; description;
    waitFor: 'click' | 'input' | 'timer' | 'none';
    codeSnippet?: string;
  }[] }[];
}
```

### 5.2 现行 Schema（D4 转向后）

```typescript
interface TutorialPackage {
  id: string;                    // 改：不再枚举限定
  meta: {
    title: string;
    description: string;
    language: string;            // 沙箱语言模式
    defaultCode: string;         // 默认代码模板
  };
  chapters: {
    id: string;
    title: string;
    steps: {
      id: string;
      title: string;
      description: string;                // MDX 讲解
      files?: Record<string, string>;     // 该步初始代码（Sandpack 多文件）
      solution?: Record<string, string>;  // 答案代码（R10）
    }[];
  }[];
}
```

### 5.3 三处改动的理由（均非随意改动）

| 改动 | 理由 |
|---|---|
| 删 `selector` | R3 已砍，无高亮目标 → 死字段 |
| 删 `waitFor` | R4 已砍，不阻塞 → 死字段 |
| `codeSnippet?: string` → `files?: Record<string,string>` | Sandpack 是**多文件**沙箱（App.jsx + index.js + ...），单字符串装不下 |
| 新增 `solution?` | R10「给我看答案」，实测自 Vue3 教程 |
| `id: 'react'\|'vue'\|'javascript'` → `id: string` | **原 Schema 硬伤**：枚举写死三个主题，加第四个就得改类型定义 = 改引擎，**与 R8「新增 JSON 无需改引擎」直接冲突** |

## 六、范围边界

### 原型阶段（阶段②）要做
- R1 内容展示、R2 步骤引导、R6 代码沙箱壳、R7 响应式、R9 shadcn 灰阶、R10 给我看答案
- 布局对标 Vue3：左 MDX 讲解 + 右沙箱（编辑器/预览上下分栏）+ 底部步骤导航

### 原型阶段**不要做**（D4 后）
- ❌ 引导蒙层 / 聚光灯 / 高亮框 / waitFor 阻塞 —— **R3/R4/R5 已砍，不得再出现**
- ❌ 深色仪表盘配色（#4C6FFF / #22D3EE / #FDBA33）—— **D5 已改 shadcn 灰阶**

### 明确不做（待用户确认，非客户原话，属推断）

- ❌ Vue / JavaScript / 算法主题的实际内容（只保证数据结构可扩展，不产出内容）
- ❌ 代码沙箱的真实执行（是否需要 WebContainer/Sandpack 级别的运行时未明确，见「七、待确认」）
- ❌ 62 篇笔记的全量搬运（原型只需样本章节）

## 七、已决策项（2026-07-17 Owner 拍板，见 `99-会议与决策/`）
