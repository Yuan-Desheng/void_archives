## 二、引导引擎（R3/R4/R5）—— 真实跑通，非视觉摆拍

`guided-tour__tour-overlay.html` 内含 446 行引擎脚本、`const steps` 5 步状态机（`state.index` 驱动），
覆盖 `waitFor: none / click / input / timer` 四态 + 一步「元素缺失降级」演示。

实测证据链：

| 测试 | 结果 |
|---|---|
| 聚光灯定位精度 | ring `201×98` 套住 target `181×78` —— 每边正好 10px，与配置 `spotlightPadding: 10` **精确吻合**（是算出来的，不是画的） |
| `waitFor:'click'` 阻塞 | 进入 step2 后「下一步」按钮 `disabled=true`；**强行连点两次无法推进**，标题保持不变 ✅ |
| 完成动作后推进 | 点 `#host-run-btn` → 提示条转「已完成，即将自动进入下一步」→ 2s 后**自动落到 step3**，并正确切入 input 阻塞态 ✅ |
| resize 重算 | 1440 → 375 后聚光灯仍套住目标（ring w=325 / target w=309）✅ |
| 降级不崩 | 引擎内置 `degrade-demo` 步骤，目标元素不存在时静默隐藏蒙层 + 保留提示条 + 允许跳过 ✅ |
| Console | 仅 favicon 404 + Tailwind CDN 生产警告，**无实质报错** |
| 死链 | 18 个页面互链 **0 死链** ✅ |

> 📌 更正记录：初次探测 `window` 上找不到配置对象，一度误判为「只有视觉、没有引擎」。
> 实为 `steps` 是模块作用域 `const`、不挂 window —— **探测方法错误，非原型缺陷**。已实测推翻该误判。

## 三、R8 配置驱动 —— 未被证明（原型阶段的正常局限，但要在设计阶段补上）

`steps` 数组**硬编码在 HTML 内的 `const` 里**，不是外部 JSON 加载。
→ 原型证明了「引擎能按配置跑」，**没有证明**「换一个 JSON 就换一个主题、引擎零改动」。
→ 这是原型形态（纯静态 HTML）的天然局限，不算 AI 偷工。
→ **但 R8 是本项目成败判据**（见 `开发计划.md` 一），必须在**阶段③设计**中以架构方式回答，
   并在阶段④开发时用「新增 `vue.json` 跑通」作为验收用例。

## 四、375px 溢出 4px —— 非真实缺陷

元凶：`SELECT#stepJump` + 其 `LABEL`（原型自带的「跳到第几步」**调试控件**），right=379 > 375。
→ 该控件是原型演示脚手架，**产品 UI 中不存在** → 不算 R7 违规，不打回。
→ 记录在案：真实产品的 375px 验收，须在阶段④重测。
