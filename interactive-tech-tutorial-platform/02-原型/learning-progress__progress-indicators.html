维护模块：学习进度与账户

本页是组件视觉/交互规范文档，非最终用户可见的业务页面。 下方 4 个组件（目录树完成标记、章节进度条、教程包全局进度、同步状态角标）在真实业务页中的落地示例见 章节/步骤目录树、 教程学习工作台、 我的学习进度。 本页的目录树与进度条 3 个演示区共享同一份 demo 数据源，点击任一步骤完成后可观察三者联动更新，用以验证实现是否一致。
数据字段 A · 目录树完成标记 B · 章节进度条 C · 教程包全局进度 D · 同步状态角标 关键交互规则

数据字段

以下字段由服务端进度存储与前端本地缓存共同维护，驱动本文档下述全部组件的渲染。

字段 类型 说明 示例值
stepId string 步骤唯一标识，格式 {chapterId}-{index} "c1-s3"
stepStatus enum 步骤完成状态：completed 已完成 / current 当前所在 / incomplete 未完成 "current"
chapterCompletionRate number 0–1 章节内已完成步骤数 / 章节总步骤数，渲染时四舍五入取整为百分比 0.67
tutorialCompletionRate number 0–1 教程包内已完成步骤数 / 教程包总步骤数，驱动全局进度指示 0.43
syncStatus enum synced 已同步 / syncing 同步中 / error 同步失败 "error"
lastSyncedAt ISO8601 | null 最近一次成功同步到服务端的时间；null 表示仅存在于本地、尚未成功同步过 "2026-07-17T09:42:15+08:00"

A · 目录树节点完成标记

三态：completed / current / incomplete

用于章节/步骤目录树中每个节点的完成状态展示。点击当前所在步骤即模拟"完成"，节点乐观打勾，不等待服务端确认；已完成步骤仍可点击回看内容。落地示例见 章节/步骤目录树。

B · 章节级进度条

D · 同步状态微型角标

syncStatus + lastSyncedAt

标记本地进度与服务端的同步情况，hover 展示已同步时间，失败态可点击重试。步骤完成后会自动触发一次同步尝试（见下方联动角标）。落地示例见 我的学习进度。

关键交互规则

以下规则适用于全部引用本组件的业务页面，实现方不应擅自变更。

乐观更新，不阻塞交互

步骤完成时目录树节点立即打勾、进度条立即增长，不等待服务端确认；服务端确认失败也不回滚已完成标记，仅将 syncStatus 置为 error 并允许重试。

同步失败可点击重试

syncStatus 为 error 时角标变为 danger 色，点击触发重新同步，进入 syncing 态；重试成功后立即更新 lastSyncedAt 并恢复 success 色。

完成 ≠ 锁定

点击已完成步骤仍可跳转回看该步骤内容；完成标记只用于展示学习进度，不会禁用点击、不会阻止用户重新查看讲解或重跑沙箱代码。

同一数据源驱动多组件

stepStatus 变更需同时驱动目录树、章节进度条（chapterCompletionRate）与全局进度（tutorialCompletionRate）重新计算，三者不应出现数值不一致。
