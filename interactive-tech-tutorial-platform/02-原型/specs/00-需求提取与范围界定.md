## 一、产品定位

**一个内容与引擎分离的交互式技术教程平台**：教程内容以 `TutorialPackage`（JSON/MDX）配置驱动，引擎负责步骤引导、聚光灯高亮与等待用户操作，配合可真实运行的代码沙箱，让学习者边读边动手。首个主题为 React，数据结构预留 Vue / JavaScript / 算法等多主题扩展。

## 二、建设背景概述

- 用户已积累 62 篇 React 学习笔记（`小满zs-react.canvas`，8 大分类：入门 3 / 工具 2 / 原理 2 / 组件 7 / CSS 方案 3 / Hooks 17 / Router 24 / Zustand 5），但以孤立 Markdown 存在，**canvas 无 edge、无顺序**，只能读、不能练，学习闭环断裂。
- 对标 Vue3 官方互动教程（cn.vuejs.org/tutorial）与 system-builder.aura.build/docs 的体验：**左讲解 / 右沙箱 + 步骤引导**已被验证是技术教学的有效形态，但市面产品把内容与引擎耦合，换主题即重写。
- 项目性质为**内部工具 / 自用教具**，无合同、不对外交付，按研发规范 §3.4 走简化流程（免立项、仅交付评审），但需完整跑通「需求→原型→设计→开发→测试→交付」链路——本项目同时是规范落地的第一块试验田。
- 因此建设重点不在内容数量，而在**引擎的可复用性**：一次把引导引擎做对，之后任何主题只是新增一个教程包。

## 三、建设目标

1. **内容与引擎分离**：新增一个 `TutorialPackage` 即可跑通新主题，不改引擎代码（对应 R8 验收口径）。
2. **完整的引导体验**：步骤导航 + 进度指示 + selector 高亮 + 聚光灯遮罩 + 等待用户操作（点击/输入）后自动推进。
3. **可真实运行的代码沙箱**：采用 Sandpack（决策 D1），学习者代码可实时预览，而非只读代码展示。
4. **承载既有 React 笔记**：以 62 篇笔记为首个主题内容源，原型阶段用 1–2 章真实内容做拟真数据。
5. **全端可用**：375px / 768px / 1440px 三档布局无横向滚动、无重叠（R7）。
6. **进度可跨端同步**：引入账户体系与服务端进度存储（决策 D3，范围主动扩大）。

## 四、用户角色

| 角色 | 说明 | 主要诉求 |
|---|---|---|
| 学习者（游客） | 未登录访问者 | 直接开始学习，进度暂存本地，不被登录挡住 |
| 学习者（登录用户） | 已登录用户 | 跨设备接着上次学，看到自己的完成率 |
| 教程作者 / Owner | 崔战伟本人，编写教程包 | 定义章节顺序与 steps，验证教程包能被引擎正确渲染与引导 |

> 说明：原型阶段不设管理员角色——单人自用，无用户管理、无审核流。

## 六、不纳入范围（outOfScope）及理由

| 内容 | 不纳入的理由 |
|---|---|
| 技术选型本身（Next.js 15 / TS / Tailwind v4 / shadcn / Drizzle / Vercel） | 实现层决策，不是界面，属阶段③设计范围 |
| PostgreSQL 表结构、Route Handler 接口契约、Clerk 接入配置 | 后端契约，原型只表达界面上的"已登录/进度已同步"，不设计存储 |
| Sandpack bundle 体积、首屏性能、移动端运行时开销 | 非功能性指标，无法落地为界面；进设计/测试阶段 |
| iframe 探针注入 / postMessage 双向通信 / 跨域坐标换算 | D2 已砍：引导只作用于 host 平台 UI，聚光灯**不追**学习者在沙箱里写出的元素 |
| Schema 的 `scope` 字段 | D2 明确不加，保持提示词原样 |
| 62 篇笔记全量搬运 | 原型只需 1–2 章样本；全量属内容工程，非原型 |
| Vue / JavaScript / 算法主题的实际内容 | R8 只要求结构可扩展，不产出内容；原型用占位主题卡表达可扩展性 |
| 62 篇笔记 → steps 的转换工艺（Q3：人工 / AI 批量 / 直接 MDX） | 未决项，是全项目最大成本项，不阻塞原型但阻塞开发 |
| 章节顺序的最终排定（Q4） | 未决项，原型自行给出一版合理顺序作为拟真数据，不作为结论 |
| 教程包作者后台（在线编辑 / 导入 / 校验教程包） | 需求未提，单人自用直接改 JSON 即可；新增即过度设计 |
| 用户管理、权限、审核流、评论 / 社区 | 需求未提，内部自用无此场景 |
| 立项/评审流程、规范章节引用、工期与人力 | 项目管理叙述，非产品功能 |

## 七、功能模块总览

| key | 模块 | 职责 | 覆盖需求 |
|---|---|---|---|
| `tutorial-catalog` | 教程与章节目录 | 主题包选择/切换、章节步骤目录、整体完成度概览，体现"新增教程包即新增主题" | R1 R2 R8 |
| `lesson-reader` | 课程讲解与步骤导航 | MDX 讲解渲染、代码块高亮、上一步/下一步、进度指示、断点续学入口 | R1 R2 |
| `guided-tour` | 引导引擎 | selector 高亮、聚光灯模糊遮罩、waitFor 等待用户操作后自动推进、异常降级 | R3 R4 R5 |
| `code-sandbox` | 代码沙箱 | Sandpack 多文件编辑 + Run + 实时预览、语言模式切换、codeSnippet 注入与重置、移动端降级 | R6 D1 |
| `learning-progress` | 学习进度与账户 | 登录入口与用户态、我的学习/完成率、步骤完成标记、跨端同步状态提示 | D3 R2 |

> 横切要求（不单列模块，由各模块共同满足）：R7 响应式三档、R9 视觉主题与明暗模式。

## 八、视觉风格判定

**styleHint = `dashboard`**。系统调性为现代开发者工具 / 交互式文档，双栏工作区（讲解 + 实时沙箱）、暗色优先、强调实时反馈与高亮聚焦，与政企审批类（gov）的表单/流程调性不符。视觉参考：cn.vuejs.org/tutorial、system-builder.aura.build/docs。
