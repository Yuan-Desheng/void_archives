# 引导引擎（guided-tour）功能定义

## 0. 模块概述

**职责边界**：引导引擎是一个与教程内容完全解耦的"运行时"。它读取一份 `TutorialPackage`（章节、步骤、selector、waitFor 等配置数据），负责：

1. 按当前步骤的 `selector` 在**平台自身 UI**（讲解区、目录树、沙箱等真实 DOM 节点）上定位目标元素并高亮；
2. 叠加"聚光灯遮罩"——背景模糊/变暗，目标元素区域清晰镂空；
3. 按步骤的 `waitFor` 类型阻塞流程，等待学习者完成指定动作（点击 / 输入 / 定时 / 无需等待）后自动推进到下一步；
4. 目标元素在 DOM 中不存在（未渲染、已被移除、selector 写错）时**优雅降级**——跳过高亮与遮罩，不阻塞、不崩溃，同时给出可诊断的信号。

**不属于引导引擎职责**（明确划清，避免范围蔓延）：
- 不解析 MDX/Markdown 内容渲染（讲解区渲染是内容层职责）；
- 不管理 Sandpack 沙箱内部逻辑（引擎只负责"引导用户去操作沙箱"，不负责沙箱本身怎么跑代码）；
- 不决定教程包内容取什么、有多少章节（内容与引擎分离的核心边界）；
- 不做用户账户认证逻辑（登录态由外部提供，引导引擎只读取"是否登录/进度"作为输入）。

**核心设计原则**：引擎是"通用状态机 + 定位/遮罩/等待"三件套的组合，任何新增主题（Vue / JavaScript / 算法）只需提供符合 `TutorialPackage` 结构的新配置数据，引擎代码零改动即可跑通新主题。

## 1. 用户故事

| # | 角色 | 故事 |
|---|------|------|
| U1 | 学习者（游客/登录用户） | 作为学习者，我打开一个教程步骤时，希望页面上被讲解的那个 UI 元素被清晰高亮出来，周围其他内容变模糊，这样我一眼就知道"现在该看哪里"。 |
| U2 | 学习者 | 作为学习者，当某一步要求我"点击目录树的某一项"时，我希望在我真的点击之前，引导不会往下走，点击后立刻自动跳到下一步，不需要我再手动点"下一步"。 |
| U3 | 学习者 | 作为学习者，当某一步只是纯讲解、不需要我做任何操作时，我希望能自己点"下一步"随时推进，不被卡住。 |
| U4 | 学习者 | 作为学习者，当某一步要求我在沙箱里输入代码时，我希望引擎能识别"我输入了内容"这件事，并据此推进，而不是要求我逐字符匹配。 |
| U5 | 学习者 | 作为学习者，当我调整浏览器窗口大小或旋转设备时，我希望聚光灯遮罩的高亮框跟着目标元素重新对齐，而不是"糊在原地"。 |
| U6 | 学习者 | 作为学习者，当某一步引导指向的 UI 元素因为我的操作、网络延迟或该元素本身还没渲染而暂时找不到时，我希望引导不崩溃、不报错弹窗打断我，而是安静跳过高亮，让我仍能通过点"下一步"手动继续。 |
| U7 | 教程作者/Owner | 作为教程作者，我希望只需要在 TutorialPackage 里配置 selector 和 waitFor，不用碰引擎代码，就能让新的一步"绑定"到平台任意 UI 元素上。 |
| U8 | 教程作者/Owner | 作为教程作者，我希望当我配置的 selector 写错（比如页面重构后 class 名变了）时，能在开发/预览环境下看到明确的告警，而不是等学习者反馈"卡住了"才发现。 |
| U9 | 学习者（登录用户） | 作为登录用户，我希望我做到第几步是被记住的，下次回来能从断点继续，而不是每次都从头点一遍。 |

## 2. 功能点清单

### 2.1 步骤定位与高亮（selector 高亮）
- 引擎按当前步骤配置的 `selector`（CSS selector 字符串）在真实 DOM 中查找目标元素。
- 查找成功：为目标元素计算包围盒（getBoundingClientRect），驱动聚光灯遮罩定位。
- 查找失败（元素不存在/未渲染）：触发**降级路径**（见 3.5 业务规则），不抛出未捕获异常。
- 支持 selector 指向的元素**动态出现**的场景（如目录树展开后才渲染的子节点）：引擎在轮询/MutationObserver 容忍窗口内重试定位，超时后降级。

### 2.2 聚光灯遮罩
- 全屏遮罩层，背景做模糊 + 半透明遮盖处理。
- 目标元素区域"镂空"（清晰可见 + 不受模糊影响），镂空框带高亮描边。
- 遮罩层不可截获目标元素区域内的点击（点击穿透到真实 UI），但遮罩层其余区域可选择性阻止误触其他 UI（由 waitFor 类型决定是否需要"锁定"）。
- 窗口 `resize` / 内容滚动 / 目标元素自身尺寸变化时，重新计算并重绘高亮框位置（防抖处理，避免拖拽窗口时高频重排）。
- 降级态（元素缺失）：遮罩不渲染或渲染为"无镂空"的纯提示态（不遮挡整页操作），并给出轻量文案提示（如"当前步骤引导的元素未找到，可直接点击下一步继续"）。

### 2.3 等待机制（waitFor）
支持四种等待类型，决定该步骤是否阻塞、以何种条件自动推进：

| waitFor 类型 | 行为 |
|---|---|
| `click` | 阻塞直到用户点击 selector 指向的目标元素（或其内部）→ 自动推进到下一步 |
| `input` | 阻塞直到用户在目标输入元素（input/textarea/代码编辑器等）中产生内容变更（非空输入或值变化）→ 自动推进 |
| `timer` | 阻塞指定时长（步骤配置 `duration`，单位毫秒/秒）后自动推进，期间学习者也可手动点"下一步"提前跳过 |
| `none` | 不阻塞，纯讲解型步骤，学习者随时可点"下一步"手动推进 |

- 除 `none` 外的等待类型，在**目标元素缺失（降级态）**时自动放宽为"允许手动点下一步推进"，避免学习者被无法满足的等待条件卡死。
- 等待达成后的"自动推进"需给出短暂视觉反馈（如高亮框成功动效）再跳转，避免用户困惑"发生了什么"。
- 用户始终保留"上一步"按钮的操作权（除首步外），阻塞态不影响后退。

### 2.4 步骤导航与进度指示
- 上一步 / 下一步按钮；下一步按钮在 `waitFor` 阻塞未满足且非降级态时禁用（视觉上标注原因，如"完成左侧操作后可继续"）。
- 当前步 / 总步数展示（如 "3 / 12"）。
- 章节内进度条（已完成步数占比）。
- 章节 / 步骤目录树：可点击跳转到**任意已解锁**步骤（已完成或当前步骤之前的步骤视为已解锁，未来步骤按教程包配置决定是否允许跳跃或强制线性）。

### 2.5 降级与容错（元素缺失不崩）
- Selector 查找失败时的完整降级链：重试（容忍动态渲染）→ 仍失败则跳过高亮/遮罩 → 记录一次诊断事件（用于日志/开发环境告警）→ 该步骤的 waitFor 自动放宽为可手动跳过。
- 引擎任何单步的渲染/定位异常都不应导致整个引导流程/整个页面白屏，异常应被捕获并局部降级。

### 2.6 教程包驱动（内容-引擎分离契约）
- 引擎仅依赖 `TutorialPackage` 的结构化数据驱动渲染，不硬编码任何具体主题的文案、selector 或 UI 结构。
- 切换教程包（如从 React 切到 Vue）等价于替换配置数据源，引擎运行时逻辑不变。

### 2.7 学习进度记录
- 步骤完成状态标记（已完成 / 当前 / 未解锁）。
- 断点续学：登录用户重新进入教程时，引擎读取服务端存储的进度，定位到上次停留的步骤。
- 章节完成率计算与展示。
- 游客态进度仅本地会话有效（不跨端同步），登录后进度写入服务端并可跨端同步（账户体系与存储属于外部依赖，引导引擎只负责读取/上报进度事件，不实现存储本身）。

## 3. 业务规则

### 3.1 waitFor 与推进的判定规则
1. 一个步骤的"可推进"状态 = `waitFor` 条件已满足 **或** `waitFor === 'none'` **或** 目标元素处于降级态（缺失）。
2. `timer` 类型的倒计时在步骤进入时启动，若用户在倒计时结束前手动点击"下一步"，视为提前满足，倒计时清除。
3. 用户来回切换步骤（点上一步再点下一步）时，`timer` 类型重新计时；`click`/`input` 类型的完成状态不做跨会话持久化判定（每次重新进入该步骤都需要重新触发一次，除非产品后续决定"已完成步骤"免重复操作——**当前范围内已完成步骤仅影响解锁状态，不免除重新查看时的等待条件**）。

### 3.2 Selector 定位规则
4. Selector 查找采用「初次查找 → 未命中则在容忍窗口内轮询/监听 DOM 变化重试 → 仍未命中判定为缺失」的流程，容忍窗口时长为固定的短时间阈值（保证不会让用户感觉"卡顿"）。
5. 一个步骤只绑定一个主 selector；若该 selector 匹配到多个 DOM 节点，取第一个匹配节点。
6. Selector 缺失判定为"缺失"而非"错误"——不区分是配置写错还是该元素在当前场景本就不存在（如响应式布局下某控件在移动端被隐藏），统一走降级路径。

### 3.3 聚光灯遮罩规则
7. 遮罩层的模糊/变暗效果不应阻挡对讲解内容区域本身的阅读（若讲解区与高亮目标同时在视口内，需保证讲解文字可读）。
8. 高亮框与目标元素之间保留统一的视觉留白（padding），不做像素级贴边。
9. Resize/滚动触发的重算需防抖，避免连续触发时的性能抖动或视觉闪烁。

### 3.4 降级与告警规则
10. 面向学习者：降级态**永远不阻塞、不报错弹窗**，最多是一条低干扰的提示文案 + 允许手动跳过。
11. 面向教程作者/Owner（开发或预览模式下）：Selector 缺失需要有可见的诊断信号（如控制台告警、预览态角标），便于在配置阶段发现问题，而不是等学习者反馈。

### 3.5 目录树跳转规则
12. 已完成或当前步骤之前的步骤 = 已解锁，可自由跳转查看。
13. 当前步骤之后的步骤是否允许跳跃访问（自由探索 vs 强制线性完成）由 TutorialPackage 的章节/步骤配置项决定，引擎需同时支持这两种模式，不写死策略。

### 3.6 响应式规则
14. 三档布局（375 / 768 / 1440）下引导蒙层与高亮框的定位计算需分别适配（尤其移动端"讲解/沙箱切换"场景下，若目标元素在当前未激活的视图内，视为不可见 → 走降级或提示用户切换视图）。

### 3.7 进度同步规则
15. 游客态操作不写服务端，登录后的操作才触发进度上报；游客中途登录，本地已完成的步骤状态尽力合并到服务端进度（不强制要求，属于账户体系模块职责，引导引擎仅暴露"当前完成状态"读写接口）。

## 4. 页面清单（每页职责）

| 页面 key | 页面名称 | 职责 |
|---|---|---|
| `tour-overlay` | 引导蒙层与聚光灯（全局叠加层） | 承载聚光灯遮罩、高亮框、当前步骤的悬浮讲解卡片（含步骤文案摘要、上一步/下一步、等待态提示），叠加在平台任意页面之上，是引导引擎最核心的可视化载体 |
| `tutorial-workspace` | 教程学习工作台（左讲解/右沙箱主界面） | 承载"左侧 MDX 讲解 + 右侧 Sandpack 沙箱"主学习界面，是引导引擎高亮定位、waitFor 监听的宿主页面 |
| `chapter-toc` | 章节/步骤目录树 | 展示教程的章节-步骤层级结构、完成度标记、已解锁/未解锁状态，支持点击跳转，是引导流程的导航入口 |
| `tutorial-catalog` | 教程包（主题）选择页 | 展示可选主题/教程包列表（标题、描述、章节数、完成度），验证"内容-引擎分离"契约的入口页 |
| `my-progress` | 我的学习进度页 | 登录用户查看跨教程包的整体学习进度、断点续学入口、跨端同步状态展示 |

> 说明：`tour-overlay` 是引导引擎的**核心自研产物**（高亮/遮罩/等待逻辑全部在此落地），其余四页是引导引擎的**宿主/联动页面**（引擎在这些页面上定位元素、驱动导航与进度展示），并非引擎本身重新发明这些页面的业务逻辑。

## 5. 关键数据字段

### 5.1 TutorialPackage（教程包，引擎的输入契约）
| 字段 | 类型 | 说明 |
|---|---|---|
| `packageId` | string | 教程包唯一标识（如 `react-basics`） |
| `title` | string | 主题标题 |
| `description` | string | 主题简介 |
| `chapters` | Chapter[] | 章节列表 |

### 5.2 Chapter（章节）
| 字段 | 类型 | 说明 |
|---|---|---|
| `chapterId` | string | 章节唯一标识 |
| `title` | string | 章节标题 |
| `steps` | Step[] | 步骤列表 |
| `navigationMode` | enum `linear` \| `free` | 该章节内后续步骤是否强制线性解锁 |

### 5.3 Step（步骤，引导引擎的核心驱动单元）
| 字段 | 类型 | 说明 |
|---|---|---|
| `stepId` | string | 步骤唯一标识 |
| `title` | string | 步骤标题 |
| `content` | string (MDX/Markdown) | 讲解内容 |
| `selector` | string \| null | 目标 UI 元素的 CSS selector，`null` 表示该步骤无高亮目标（纯讲解） |
| `waitFor` | enum `click` \| `input` \| `timer` \| `none` | 等待类型 |
| `waitForTarget` | string \| null | waitFor 为 click/input 时的具体监听目标 selector（可与 `selector` 相同或不同） |
| `duration` | number \| null | waitFor 为 timer 时的等待时长（毫秒） |
| `codeSnippet` | object \| null | 注入沙箱的代码片段（多文件），null 时沙箱保持默认模板 |
| `spotlightPadding` | number | 高亮框相对目标元素的留白（像素，可选，有默认值） |

### 5.4 TourRuntimeState（引擎运行时状态，非配置数据）
| 字段 | 类型 | 说明 |
|---|---|---|
| `currentChapterId` / `currentStepId` | string | 当前所在位置 |
| `elementStatus` | enum `located` \| `degraded` \| `pending` | 当前步骤 selector 的定位结果 |
| `waitSatisfied` | boolean | 当前步骤的 waitFor 条件是否已满足 |
| `completedSteps` | string[] | 已完成的 stepId 集合 |
| `unlockedSteps` | string[] | 已解锁可跳转的 stepId 集合 |

### 5.5 学习进度（服务端存储，引擎读写的外部数据）
| 字段 | 类型 | 说明 |
|---|---|---|
| `userId` | string \| null | 登录用户标识，游客为 null（本地态） |
| `packageId` | string | 关联的教程包 |
| `lastStepId` | string | 断点续学定位 |
| `completedStepIds` | string[] | 已完成步骤集合 |
| `chapterCompletionRate` | Record<chapterId, number> | 各章节完成率 |
| `updatedAt` | datetime | 最近更新时间（跨端同步依据） |

## 6. 异常 / 边界场景

| 场景 | 处理方式 |
|---|---|
| Selector 指向的元素在 DOM 中不存在 | 走降级路径：不渲染高亮/遮罩，允许手动点下一步，开发态告警（见规则 4、10、11） |
| Selector 指向的元素存在但不在当前视口/被其他元素遮挡 | 高亮框正常计算位置，若不在视口内需自动滚动使其可见（scrollIntoView），滚动失败也不阻塞 |
| Selector 匹配到多个节点 | 取第一个匹配节点，不报错 |
| 响应式切换后（如 1440→375）目标元素被 CSS 隐藏 | 视为"不可见"降级，移动端场景下提示用户切换到"讲解/沙箱"对应视图 |
| 用户在 waitFor=click 步骤疯狂连续点击 | 首次点击即判定满足并推进，后续点击不重复触发（防止重复推进多步） |
| 用户在 waitFor=input 步骤输入后又清空 | 以"曾经产生过一次有效输入"为满足条件即可推进（不要求推进时刻仍非空），避免用户因删除重输而卡住；若产品希望更严格判定可在教程包配置中扩展（当前范围内取宽松策略） |
| waitFor=timer 但用户中途离开页面/切后台 | 计时器暂停或按实际经过时间容错处理，避免切回后瞬间跳过多步（具体依赖浏览器 visibility 事件，属于实现细节，此处仅约束"不应导致进度错乱"） |
| resize 极高频触发（如拖拽窗口边缘） | 防抖节流，避免遮罩计算成为性能瓶颈或出现高亮框抖动 |
| 教程包配置本身缺失必填字段（如 step 无 stepId） | 引擎需容错跳过该异常步骤并告警，不因单个脏数据步骤导致整个章节/教程包渲染失败 |
| 游客态学习进度与登录后服务端进度冲突 | 以"并集/取较新"的宽松合并策略处理（具体合并算法属于进度存储模块职责，引导引擎只需正确读取合并后的结果） |
| 章节导航模式为 `linear` 但用户直接改 URL 跳转到未解锁步骤 | 引擎读取到目标步骤未解锁时，重定向回该章节最后一个已解锁步骤，并给出提示 |
| 同一页面同时存在多个可能匹配 selector 的组件实例（如目录树被复用在移动端抽屉和桌面侧栏两处） | 按规则 5 取第一个匹配节点；教程包作者需保证 selector 唯一性以避免该场景，引擎不做去重智能判断 |
| 沙箱（Sandpack）尚未完成初始化加载完成时，步骤已指向沙箱内部 selector | 视为元素暂未渲染，走"容忍窗口内重试"路径（规则 4），沙箱加载完成后正常定位 |

## 7. 验收关注点（供测试/设计阶段参考，非本文档强制范围）

- 375 / 768 / 1440 三档下聚光灯遮罩与高亮框定位无错位、无横向滚动。
- 明暗模式下遮罩模糊/变暗效果、高亮描边对比度均可辨识。
- 快速连续切换步骤（含浏览器前进/后退）时不出现遮罩残留、状态错乱。
- 人为删除目标 DOM 节点（模拟 selector 失效）后引导不崩溃、可继续操作。
