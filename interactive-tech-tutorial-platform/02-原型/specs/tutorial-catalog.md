# 教程与章节目录（tutorial-catalog）功能定义

## 1. 模块概述

「教程与章节目录」是学习者进入互动教程系统的**入口与导航层**，不负责步骤内容的讲解渲染、聚光灯引导、代码沙箱运行等引擎能力（由「步骤引导与代码沙箱」等模块负责），只负责三件事：

1. **主题包（TutorialPackage）选择与切换** —— 以卡片/列表形式展示平台当前已配置的所有教程包（首个为 React，预留 Vue / JavaScript / 算法），每个包展示标题、简介、章节数、当前完成度；
2. **章节 / 步骤目录树** —— 进入某个教程包后，以树形结构展示该包的章节与步骤，标注解锁 / 完成状态，支持跳转到任意已解锁步骤，是学习者在长教程中定位、跳转、断点续学的主要工具；
3. **整体完成度概览** —— 聚合展示学习者在单个包内、以及跨所有教程包的学习进度，是「我的学习」页的核心内容，并承载登录态与跨端同步的界面表达。

**架构原则（本模块必须体现）**：本模块只消费 `TutorialPackage` 配置对象的**结构化数据**（元信息 + 章节/步骤树），不硬编码任何主题相关的展示逻辑。验收标准是——工程侧在配置层新增一个 `TutorialPackage`（如 Vue 教程包），本模块无需改一行代码即可在主题选择页、目录树、进度概览中正确渲染出新主题。本模块界面上应有意识地体现"这是由数据驱动生成的列表/树"，而不是为 React 主题单独设计视觉结构。

## 2. 用户故事

| # | 角色 | 故事 |
|---|------|------|
| 1 | 游客学习者 | 作为一个第一次访问的游客，我想看到平台上有哪些教程主题、每个主题大概讲什么、多少章节，以便挑一个开始学习 |
| 2 | 游客学习者 | 作为游客，我想在一个教程包内看到完整的章节/步骤目录，即使还没登录，也能预览目录结构，判断值不值得学 |
| 3 | 登录学习者 | 作为登录用户，我想看到自己在当前教程包的完成度（第几步/共几步、章节进度条），以及上次学到哪一步，点一下就能续学 |
| 4 | 登录学习者 | 作为登录用户，我想在目录树里自由跳转到任意"已解锁"的步骤复习，而不必每次都从头一步步点 |
| 5 | 登录学习者 | 作为已在多端学习的用户（手机看、电脑练），我想登录后进度自动同步到最新一端，不用重新学 |
| 6 | 登录学习者 | 作为学习者，我想有一个"我的学习"总览页，一眼看到自己学过几个主题、总体完成了多少 |
| 7 | 教程作者/Owner | 作为作者，我新增一个 TutorialPackage 配置后，希望它能原样出现在主题选择页和目录树里，不需要求助研发改代码 |
| 8 | 教程作者/Owner | 作为作者，我想在调试阶段无视"解锁"限制，直接跳到任意步骤检查内容和顺序是否正确 |

## 3. 功能点清单

### 3.1 主题包选择与切换
- F1 主题包列表：以卡片形式展示所有 `TutorialPackage`（标题、简介、封面/图标、章节数、总步数、当前用户完成度百分比）
- F2 主题包状态标识：可用 / 维护中 / 内容筹备中（章节数为 0），维护中与筹备中置灰不可进入
- F3 主题包切换：学习者在任意教程包内可通过顶部/侧边入口返回主题选择页，切换到另一个包，各包进度互不影响、各自独立记录
- F4 "继续学习"入口：对已有进度的包，卡片上显示"继续学习"并直达上次学到的步骤；无进度的包显示"开始学习"直达第一章第一步

### 3.2 章节 / 步骤目录树
- F5 树形目录渲染：按 `TutorialPackage.chapters[].steps[]` 结构渲染两级（章节 → 步骤），支持章节折叠/展开
- F6 步骤状态图标：未解锁（锁形）、已解锁未完成（空心圆）、进行中/当前（高亮）、已完成（对勾）
- F7 跳转导航：点击任意"已解锁"或"已完成"步骤，跳转到对应步骤详情（由引导引擎接管渲染）；点击"未解锁"步骤给出提示，不跳转
- F8 章节级进度条：每个章节标题旁展示"已完成步骤数/章节总步数"及百分比进度条
- F9 当前位置高亮 + 自动滚动定位：进入目录树时自动展开并滚动到"当前学习到的步骤"
- F10 大纲搜索/定位（辅助）：教程包步骤数较多（如 React 包源自 62 篇笔记、8 大分类）时，目录树顶部提供分类/关键字快速过滤

### 3.3 整体完成度概览
- F11 单包完成度：教程包详情页顶部展示总体完成度环形图/进度条（已完成步骤数/总步数）
- F12 「我的学习」总览页：登录用户可查看所有已开始学习的教程包列表，每个包展示完成度、最近学习时间，点击直达断点
- F13 登录入口与用户态展示：游客态显示"登录以保存进度"提示；登录态显示用户身份与同步状态
- F14 跨端同步状态表达：显示"进度已同步"/"同步中"/"离线，未同步"等状态提示（本模块只做**界面表达**，实际同步逻辑由账户与进度服务模块实现）

## 4. 业务规则

1. **数据驱动约束**：主题选择页与目录树的渲染逻辑必须完全由 `TutorialPackage` 配置数组驱动（遍历渲染），代码中不得出现针对某个具体主题（如“React”）的 if/switch 分支。
2. **解锁规则（默认线性）**：同一章节内，步骤 N+1 默认需要步骤 N 标记"已完成"才解锁；章节之间同理，下一章节第一步需要上一章节最后一步完成才解锁。教程包配置可显式声明某步骤 `unlockPolicy: free` 跳过限制（用于速览型章节）。
3. **角色差异**：
   - 游客可查看主题选择页与目录树结构，但只能进入**第一章第一步**；点击其余未解锁步骤提示"登录后开始学习"；游客的学习进度仅保存在本地（localStorage），不跨端同步。
   - 登录用户按 3.1 规则解锁推进，进度实时写入服务端。
   - 作者/Owner 角色目录树中所有步骤均视为"已解锁"（不含"已完成"状态区分），用于内容自查，且该行为需要有明显的视觉标识（如"预览模式"角标），避免误以为是正常学习者视角。
4. **完成度计算**：章节完成度 = 该章节已完成步骤数 / 章节总步数；教程包完成度 = 全部已完成步骤数 / 全部步骤数；两者均实时反映最新提交结果，不做缓存滞后展示（允许短暂 loading 态）。
5. **进度归属与合并**：游客本地进度在登录那一刻，如与账号在服务端已有的进度冲突，以"更靠后的步骤"为准合并（不覆盖式，取并集里进度更深的一侧），合并后清空本地缓存，改为服务端为准。
6. **多端并发**：登录用户在多端同时学习产生冲突时，以服务端记录的"最后写入时间戳"较新的一侧为准（弱一致，无需强锁）。
7. **新增主题包零改动**：新增 `TutorialPackage` 只需新增配置（id、title、description、chapters/steps 树、封面），主题选择页、目录树组件、完成度计算逻辑均无需改动即可正确渲染与统计。

## 5. 页面清单

| 页面 | 职责 |
|------|------|
| 主题包选择页 | 平台入口，展示所有教程包卡片（标题/简介/章节数/完成度/状态），承载主题切换与登录入口 |
| 教程详情页（目录树 + 概览） | 单个教程包内的导航壳层：左侧章节/步骤目录树 + 顶部整体完成度、"继续学习"/角色态提示，是跳转进入具体步骤内容的枢纽 |
| 我的学习总览页 | 登录用户查看自己在**所有**教程包的学习足迹：完成度汇总、最近学习时间、跨端同步状态、断点续学入口 |

## 6. 关键数据字段

### TutorialPackage（教程包配置，引擎消费的数据源）
| 字段 | 说明 |
|------|------|
| packageId | 唯一标识，如 `react` / `vue` / `javascript` / `algorithm` |
| title / description | 主题标题、简介 |
| coverIcon / theme | 封面图标、主题色（用于卡片视觉区分） |
| status | `active` / `maintenance` / `draft`（筹备中） |
| chapters[] | 章节数组，每章含 chapterId、title、order、steps[] |
| chapters[].steps[] | 步骤数组，每步含 stepId、title、order、unlockPolicy |
| totalSteps | 冗余统计字段，总步数（可由 chapters 派生） |

### UserProgress（学习者进度，per 用户 per 包）
| 字段 | 说明 |
|------|------|
| userId / guestSessionId | 登录用户 ID，或游客本地会话标识 |
| packageId | 关联教程包 |
| completedStepIds[] | 已完成步骤 ID 列表 |
| currentStepId | 当前/最近学习到的步骤（断点续学锚点） |
| lastLearnedAt | 最近学习时间戳（用于多端冲突判定与“最近学习”排序） |
| syncStatus | `synced` / `syncing` / `offline`（界面表达用） |

### 目录树节点（前端渲染态，由上述数据派生）
| 字段 | 说明 |
|------|------|
| nodeType | `chapter` \| `step` |
| unlockState | `locked` \| `unlocked` \| `completed` \| `current` |
| progressPercent | 章节级/包级完成百分比 |

## 7. 异常 / 边界

| 场景 | 处理方式 |
|------|---------|
| 教程包章节数为 0（内容筹备中） | 卡片置灰、禁用点击，标注"内容筹备中" |
| 教程包 status = maintenance | 卡片置灰、禁用点击，标注"维护中"及可选原因文案 |
| 章节/步骤配置为空（章节下 steps 为空数组） | 目录树中该章节显示"暂无内容"，不计入解锁链路，不阻塞后续章节 |
| 步骤 ID 重复或树形结构成环（配置错误） | 开发环境下抛出明显的配置校验错误提示；生产环境降级为只渲染首次出现的节点，跳过重复项，不崩溃 |
| 游客点击未解锁步骤 | 弹出轻提示"登录后解锁完整教程"，附登录入口，不跳转 |
| 游客点击已解锁但服务端无进度记录的步骤（本地缓存被清） | 视为从零开始，不报错，正常进入 |
| 网络异常，无法拉取服务端进度 | 使用本地缓存的最近一次进度兜底渲染，目录树/概览页顶部提示"离线，进度可能未同步" |
| 多端并发写入冲突 | 以服务端 `lastLearnedAt` 较新一侧为准覆盖展示，另一端下次拉取时静默刷新，不弹错误 |
| 步骤/章节数量很大（如 62 篇笔记合并的长目录） | 目录树默认折叠除当前章节外的其他章节；提供分类/关键字过滤，避免整页超长滚动 |
| 375px 窄屏下目录树 | 目录树收纳为可展开的抽屉/下拉，不常驻左侧挤压主内容区，避免横向滚动 |
| 作者/Owner 预览模式下的完成度 | 完成度概览显示"预览模式，不计入学习进度"，与真实学习者进度数据隔离，不写入 UserProgress |
| 教程包被下线/删除，而用户历史进度仍存在 | 我的学习总览页对应条目标注"该教程已下线"，进度数据保留但入口置灰不可进入 |
