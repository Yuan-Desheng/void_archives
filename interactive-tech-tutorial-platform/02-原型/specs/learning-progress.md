# 模块功能定义：学习进度与账户（learning-progress）

## 0. 模块概述

「学习进度与账户」负责平台的**用户态展示与登录入口**、**学习进度的记录与呈现**、**跨端同步状态提示**三件事。它不负责教程内容渲染、步骤引导逻辑、沙箱运行（这些属于「引导引擎」「教程内容」等其他模块），只消费引擎在步骤完成/切换时抛出的进度事件，把进度**持久化**并**可视化**。

核心定位：
- 对**游客**：在合适的时机（如完成第一步、尝试跳章）提示登录可保存进度，不强制登录墙。
- 对**登录用户**：进度写入服务端，提供「我的学习」页面看全局完成情况，支持断点续学（回到平台任意入口都能一键回到上次学到的位置）。
- 对**教程作者/Owner**：本模块不提供作者管理界面（作者管理属于教程包/内容管理模块），但进度统计口径需与教程包的章节/步骤结构对齐，供后续可能的学习数据分析复用。

本项目为内部工具（按规范 §3.4 简化流程），登录方式、进度同步只需覆盖单人多端最常见路径，不做企业级账户体系（无 SSO / 多角色权限 / 团队空间）。

## 1. 用户故事

1. 作为**游客学习者**，我打开教程后可以直接开始学习不被登录打断，但当我完成第一步时希望看到「登录可保存进度」的轻提示，这样我知道进度默认不会丢失是有代价的。
2. 作为**游客学习者**，我关闭浏览器再打开教程，希望本地至少还能续上未登录状态下的进度（同设备），而不是从头开始。
3. 作为**登录用户**，我在公司电脑学到第 3 章，回家用笔记本打开同一教程，希望自动定位到「上次学到」的步骤，而不用手动翻找。
4. 作为**登录用户**，我想在「我的学习」页看到所有教程包的完成度总览，知道自己学了多少、还剩多少。
5. 作为**登录用户**，我完成一个步骤后希望看到明确的「已完成」标记（如目录树打勾），这样我能确认进度确实被记住了。
6. 作为**登录用户**，我想在多章节目录里一眼看出哪些章节全部完成、哪些进行中、哪些还没开始。
7. 作为**登录用户**，我在弱网/离线环境下继续学习，希望进度先在本地保存，网络恢复后自动同步到服务端，不因网络问题丢失进度也不重复报错打断我。
8. 作为**登录用户**，我想快速看到「上次同步时间」或同步失败提示，这样我知道当前看到的进度是不是最新的。
9. 作为**新用户**，我想用最少的步骤完成登录（如邮箱验证码/第三方登录），因为这只是个学习工具，不想为登录设置复杂密码。
10. 作为**已登录用户**，我想方便地退出登录（如切换账户排查问题），退出后本模块应清晰提示当前是游客状态。

## 2. 功能点清单

| 编号 | 功能点 | 说明 |
|---|---|---|
| F1 | 用户态展示（Header/导航区） | 未登录显示「登录」入口；已登录显示头像/昵称 + 下拉菜单（我的学习、退出登录） |
| F2 | 登录入口与登录弹窗/页 | 提供邮箱验证码登录（简化流程，无需密码），登录成功后原地刷新用户态，不跳转丢失当前学习上下文 |
| F3 | 游客进度本地保存 | 未登录时步骤完成状态写入 localStorage，登录后可选择合并到账户 |
| F4 | 登录态进度合并提示 | 游客首次登录时，若本地已有进度，弹出提示「是否合并本地学习记录」 |
| F5 | 登录引导轻提示 | 游客完成首个步骤 / 尝试跨设备场景时，非阻断式提示登录价值（可关闭，不重复打扰） |
| F6 | 步骤完成标记 | 引擎判定 waitFor 完成后，本模块记录该 step 为已完成，UI 侧目录树/进度条同步打勾 |
| F7 | 断点续学 | 记录每个教程包的 lastVisitedStep，用户重新进入教程包时提示/自动跳转到上次位置 |
| F8 | 章节完成率计算 | 按「已完成步骤数 / 章节总步骤数」计算章节完成率；教程包完成率为章节完成率的加权平均（按各章节步骤数加权） |
| F9 | 「我的学习」总览页 | 展示当前用户所有已开始学习的教程包卡片，含总体完成度、最近学习时间、继续学习入口 |
| F10 | 跨端同步状态提示 | 展示同步状态：已同步 / 同步中 / 同步失败（含重试）/ 离线待同步，及最近一次成功同步时间 |
| F11 | 离线队列与自动重试 | 网络不可用时，完成事件先入本地队列；网络恢复后自动批量提交，失败按退避策略重试，累计失败达上限则转为「需要手动重试」 |
| F12 | 退出登录 | 清除服务端会话与本地登录态，UI 回退至游客态；本地已同步的进度不因退出而被清空（下次登录同账户仍可见） |
| F13 | 多端进度冲突处理 | 同一教程包在两端并发产生的完成记录，采用「按 step 取并集 + 按时间戳取最新 lastVisitedStep」策略合并，不做复杂三方合并 |

## 3. 业务规则

### 3.1 登录与身份
- 登录方式：邮箱 + 验证码（简化流程），验证码 5 分钟有效，60 秒内不可重复发送。
- 未登录状态下允许浏览与操作教程全部功能（含代码沙箱），仅进度落地方式不同（本地 vs 服务端）。
- 同一浏览器/设备的游客进度用一个稳定的匿名 ID（如生成并持久化在 localStorage 的 UUID）标识，避免每次访问生成新记录。
- 登录成功后，若检测到该浏览器存在游客进度且与账户已有进度不同，触发 F4 合并提示；用户可选择「合并」或「仅保留账户记录」，默认不静默覆盖。

### 3.2 进度记录粒度
- 进度的最小单元是「步骤（step）完成状态」：`未开始 / 已完成`，不做「进行中」的中间态持久化（进行中由前端会话态表达，不落库）。
- 章节完成率 = 该章节下已完成步骤数 / 该章节步骤总数，向下取整到百分比整数展示（如 66%）。
- 教程包完成率 = Σ(各章节已完成步骤数) / Σ(各章节总步骤数)。
- 已解锁但未标记完成的步骤不计入完成率分子，但计入分母。

### 3.3 断点续学
- `lastVisitedStep` 在用户每次进入某一步骤时更新（无论该步骤是否完成）。
- 用户重新打开某教程包时：若存在 `lastVisitedStep` 且不是第一步，提示「继续上次学习（章节 X · 步骤 Y）」与「从头开始」两个入口，不强制跳转。

### 3.4 跨端同步
- 同步触发时机：步骤完成事件发生时（准实时）、页面可见性恢复时（visibilitychange）、用户手动触发（如「我的学习」页下拉刷新）。
- 同步状态四态：`synced（已同步）/ syncing（同步中）/ offline_pending（离线待同步，本地队列非空且当前无网络）/ error（同步失败，已重试达上限）`。
- 离线队列采用「至少一次」语义：允许服务端收到重复完成事件，服务端按 (userId, tutorialId, stepId) 幂等去重。
- 冲突合并策略（F13）：完成状态取两端并集（只要任一端标记完成即视为完成，不可回退为未完成）；`lastVisitedStep` 取时间戳较新的一端。

### 3.5 边界与降级
- 教程包/章节/步骤结构变更（如作者增删了步骤）后，历史进度按 stepId 匹配存量记录，找不到对应 stepId 的历史记录不参与当前完成率计算（不报错、不阻断）。
- 服务端不可达时，所有登录态功能均以「最后一次成功同步的本地缓存」展示 + 明确的过期/离线提示，不允许展示空白或崩溃态。

## 4. 页面清单

| 页面 key | 页面名称 | 职责概述 |
|---|---|---|
| auth-entry | 登录入口与登录弹窗 | 承载「登录」入口按钮、用户态下拉菜单，以及邮箱验证码登录的弹窗流程 |
| my-learning | 我的学习 | 登录用户的学习总览：各教程包完成度、最近学习时间、继续学习/重新开始入口 |
| progress-indicators | 进度标记与同步状态（嵌入式） | 非独立路由页面，定义嵌入在教程学习页目录树/顶部的完成打勾、章节进度条、同步状态角标的展现规范 |

> 说明：`progress-indicators` 不是独立可跳转页面，而是「学习进度」相关 UI 组件在教程学习主界面（属于其他模块的页面）中的嵌入规范，此处单列是为了让开发/设计有明确交付对象；如实现阶段决定合并进 auth-entry 的全局状态组件文档，可在设计环节调整，不影响功能范围。

### 4.1 auth-entry（登录入口与登录弹窗）
- **职责**：任何页面顶部导航区的用户态展示；触发登录弹窗；登录成功后的合并提示；退出登录。
- **主要 UI 区块**：
  - 顶部导航用户态区（游客：登录按钮；登录后：头像+昵称+下拉菜单）
  - 登录弹窗（邮箱输入 → 发送验证码 → 验证码输入 → 提交）
  - 本地进度合并提示条/弹窗（首次登录且检测到本地进度时）
  - 登录轻提示气泡（游客完成首个步骤时，非阻断，可关闭且不重复弹）
- **关键数据字段**：`email`、`verificationCode`、`userId`、`displayName`、`avatarUrl`、`loginTimestamp`、`hasLocalGuestProgress`（bool）
- **关键交互**：点击登录 → 弹出弹窗；发送验证码（含倒计时按钮态）；提交验证码校验成功/失败反馈；登录成功后弹窗关闭并刷新导航用户态；点击「合并本地进度」/「仅保留账户记录」；点击下拉菜单「退出登录」→ 二次确认（可选，视交互复杂度）→ 回到游客态。

### 4.2 my-learning（我的学习）
- **职责**：登录用户查看自己在所有教程包中的学习总览，并从这里继续学习。
- **主要 UI 区块**：
  - 页面标题 + 总体统计条（已学习教程包数、平均完成度）
  - 教程包进度卡片列表（每卡片：标题、封面/图标、章节数、完成度环形/条形图、最近学习时间、继续学习按钮）
  - 空状态（尚未开始任何教程包时的引导文案 + 「去学习」入口）
  - 同步状态角标（页面级，展示当前账户进度的同步状态与最近同步时间）
- **关键数据字段**：`tutorialId`、`tutorialTitle`、`totalSteps`、`completedSteps`、`completionRate`、`lastVisitedAt`、`lastVisitedStepId`、`syncStatus`、`lastSyncedAt`
- **关键交互**：点击教程包卡片 → 跳转该教程包并按 F7 规则定位到断点；下拉刷新或点击「刷新」手动触发同步；同步失败时点击「重试」；空状态点击「浏览教程」跳转教程选择页（属于其他模块）。

### 4.3 progress-indicators（进度标记与同步状态嵌入规范）
- **职责**：定义步骤完成后、目录树/进度条/顶部状态角标应如何呈现「已完成」「进行中」「同步状态」，供教程学习页（其他模块页面）实现时引用。
- **主要 UI 区块**：
  - 目录树节点完成标记（打勾图标 + 已完成态样式，区分「已完成/未完成/当前所在」三态）
  - 章节级进度条（当前步/总步数 + 百分比）
  - 教程包级全局进度指示（顶部或侧栏，展示整体完成度）
  - 同步状态微型角标（图标 + hover 展示「已同步 · HH:mm」或「同步失败，点击重试」）
- **关键数据字段**：`stepId`、`stepStatus`（completed/not_completed）、`chapterProgress`（completed/total）、`tutorialProgress`（completed/total）、`syncStatus`、`lastSyncedAt`
- **关键交互**：步骤完成时目录树节点即时打勾（乐观更新，不等待服务端确认阻塞 UI）；同步失败时角标变色并可点击重试；点击已完成步骤仍可跳转回看内容（完成标记不代表锁定）。

## 5. 关键数据字段汇总

| 字段名 | 类型 | 说明 |
|---|---|---|
| userId | string | 登录用户唯一标识；游客态为空，使用 guestId 替代 |
| guestId | string (UUID) | 游客匿名标识，持久化于 localStorage |
| tutorialId | string | 教程包唯一标识 |
| chapterId | string | 章节标识 |
| stepId | string | 步骤标识 |
| stepStatus | enum: completed / not_completed | 步骤完成状态，不含中间态 |
| completedAt | timestamp | 步骤标记完成的时间 |
| lastVisitedStepId | string | 该教程包下用户最后访问的步骤，用于断点续学 |
| lastVisitedAt | timestamp | 最后访问时间 |
| chapterCompletionRate | number (0-100) | 章节完成率 |
| tutorialCompletionRate | number (0-100) | 教程包整体完成率 |
| syncStatus | enum: synced / syncing / offline_pending / error | 当前同步状态 |
| lastSyncedAt | timestamp | 最近一次成功同步时间 |
| pendingQueueLength | number | 本地待同步事件队列长度（用于展示"N 条待同步"） |

## 6. 异常与边界场景

| 场景 | 处理方式 |
|---|---|
| 未登录用户直接访问「我的学习」页 | 展示登录引导页/弹窗，不展示任何账户数据；不做静默跳转到登录页（弹窗态更轻量） |
| 验证码错误/过期 | 输入框下方内联错误提示，允许重新输入或重新发送，不清空已输入的邮箱 |
| 登录时网络失败 | 提示「登录失败，请检查网络后重试」，不吞错误、不假装登录成功 |
| 游客本地进度与账户进度冲突（合并提示未处理即关闭弹窗） | 默认按「仅保留账户记录」处理，不静默丢弃也不静默覆盖，需在下次登录时不再重复打扰同一批本地数据 |
| 教程包结构变更导致历史 stepId 失效 | 该 stepId 记录不参与当前完成率计算，不报错、不阻断学习；如可能，静默清理陈旧记录 |
| 长时间离线（本地队列持续增长） | 队列达到上限（如 500 条）后合并同类事件（同一 stepId 多次完成只保留一条），避免无限增长 |
| 同步重试多次仍失败 | 状态置为 error，角标提供手动「重试」；不无限自动重试拖累前端性能 |
| 多标签页同时操作同一教程包 | 以本地队列 + 服务端幂等去重兜底，不强制单标签页限制（内部工具，接受轻量数据竞争） |
| 用户在游客态下产生大量本地进度后才登录 | 合并提示需清晰列出"本地有 N 个教程包、M 个步骤记录"，避免用户误判数据量后误操作覆盖 |
| 服务端进度接口 5xx / 超时 | 前端展示最后一次本地缓存 + `error` 同步态，不阻塞学习页面的其他功能（沙箱、内容阅读均可用） |
| 头像/昵称加载失败 | 使用默认占位头像与邮箱前缀作为兜底昵称，不留空白 |
