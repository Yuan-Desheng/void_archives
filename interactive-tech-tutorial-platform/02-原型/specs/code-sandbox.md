# 模块功能定义：代码沙箱（code-sandbox）

## 0. 模块概述

代码沙箱是引导引擎与教程内容之间的"动手实践"载体，基于 [Sandpack](https://sandpack.codesandbox.io/) 构建，承担教程页面右侧（或移动端下半屏）的可真实运行代码环境。它与引导引擎解耦：引擎只负责在合适的步骤把 `codeSnippet` 注入沙箱、判断 `waitFor` 条件是否满足；沙箱本身不关心自己处在教程的第几步，只响应"注入代码 / 重置代码 / 读取当前代码状态"这几类外部指令。

核心能力：
- 多文件工程结构的编辑与切换（如 `App.js` + `styles.css` + `index.js`）
- 真实的浏览器端打包运行（依赖 Sandpack bundler，而非静态代码高亮展示）
- 手动 Run（而非纯 keystroke 实时编译，兼顾性能与可控性，具体见业务规则）与实时预览面板
- 语言模式（`template`）随教程包主题切换：React 主题用 `react`，未来 Vue 主题用 `vue`，JavaScript 主题用 `vanilla`
- 步骤级代码注入（`codeSnippet`）与"重置为默认代码模板"
- 375 / 768 / 1440 三档响应式布局，移动端讲解区与沙箱区可切换（非并排）

沙箱不负责：步骤导航、聚光灯高亮、waitFor 语义判断（这些是引导引擎与步骤目录模块的职责）；沙箱只暴露状态与事件供引擎消费。

## 1. 用户故事

- 作为**游客学习者**，我希望在读完一段讲解后，能在右侧沙箱里直接看到对应的示例代码已经写好，点 Run 就能看到效果，而不用自己抄代码。
- 作为**游客学习者**，我希望能修改沙箱里的代码（比如改一个 prop 的值），点 Run 后立刻看到预览变化，用来验证我对知识点的理解。
- 作为**登录学习者**，我希望改乱了代码之后，能一键"重置为默认代码"，回到该步骤本来的示例，而不用重新刷新整个页面丢失学习进度。
- 作为**登录学习者**，我希望在手机上也能看代码、跑代码，即使屏幕小，也能通过"讲解 / 代码"切换 tab 分别看清楚，而不是被挤成无法阅读的两栏。
- 作为**教程作者**，我希望某一步骤的 `waitFor` 设置为 `input`（比如"把 `count` 初始值改成 `10`"），当学习者在沙箱里编辑代码满足该条件时，引擎能收到"用户已完成该动作"的信号并自动推进到下一步。
- 作为**教程作者**，我希望不同主题的教程包能各自声明自己的语言模式（React/Vue/JS），沙箱组件本身不需要为新主题改代码，只需读取 `TutorialPackage.sandboxTemplate` 字段。
- 作为**学习者（暗色模式用户）**，我希望沙箱的编辑器主题跟随平台的明暗模式切换，不要出现"页面是暗色、代码编辑器却是刺眼的白色"的割裂体验。

## 2. 功能点清单

| 编号 | 功能点 | 说明 |
|---|---|---|
| F1 | 多文件切换 | 沙箱内以文件 Tab 形式展示当前步骤涉及的多个文件，学习者可点击切换编辑区显示的文件 |
| F2 | 代码编辑 | 基于 CodeMirror（Sandpack 内置）的可编辑代码区，支持语法高亮、基础自动缩进 |
| F3 | Run 运行 | 点击 Run 按钮（或步骤要求时的快捷键）触发 Sandpack bundler 重新打包并刷新预览面板 |
| F4 | 实时预览 | 预览面板（iframe 沙箱）展示 Run 后的渲染结果，支持基础的 console 错误提示 |
| F5 | 语言模式随主题切换 | 沙箱的 `template`（如 `react` / `vanilla` / `vue`）由当前 TutorialPackage 的 `sandboxTemplate` 字段决定，组件不写死 |
| F6 | codeSnippet 注入 | 引擎在进入某一步骤时，把该步骤配置的 `codeSnippet`（含目标文件路径与内容）写入沙箱对应文件，覆盖当前内容 |
| F7 | 重置为默认代码 | 提供"重置"按钮，将当前步骤的所有文件还原为该步骤 `codeSnippet` 声明的初始内容（非教程包全局默认，是"当前步骤"的默认） |
| F8 | 移动端布局降级 | ≤768px 时讲解区与沙箱区从左右并排改为上下 Tab 切换，避免横向挤压导致代码不可读 |
| F9 | waitFor=input 联动 | 沙箱编辑内容变化时对外广播 `onCodeChange` 事件（含文件路径、内容 diff 或简单文本比对结果），供引擎判断 `waitFor: input` 的完成条件 |
| F10 | 明暗主题联动 | 沙箱编辑器与预览面板的配色跟随平台主题（`light` / `dark`）实时切换，不需要用户单独设置 |
| F11 | 运行状态反馈 | Run 过程中展示 loading 态；编译报错时在预览区展示 Sandpack 原生错误堆栈（不吞错） |
| F12 | 依赖声明 | 每个步骤可声明 `dependencies`（npm 包名+版本），沙箱据此加载对应依赖，供示例代码 import 使用 |

## 3. 业务规则

1. **Run 触发方式**：默认采用"手动 Run 按钮触发"，不做逐字符自动编译刷新（Sandpack 默认的 auto-run 会在低性能设备上造成卡顿，且频繁重打包会打断"等待用户操作"的判断时机）。教程作者可在步骤配置里将某一步标记为 `autorun: true`（如仅做只读演示的步骤），此时首次进入即自动 Run 一次，之后仍需手动点击才会重新运行。
2. **codeSnippet 注入时机**：仅在"进入新步骤且该步骤声明了 `codeSnippet`"时注入；同一步骤内学习者手动修改代码后，不会被重复注入覆盖，除非学习者主动点击"重置"。
3. **注入范围**：`codeSnippet` 可以只声明本步骤要变化的文件（增量更新），未声明的文件保留上一步骤的最终状态；若步骤显式声明 `resetAllFiles: true`，则先清空所有文件到教程包的全局默认模板，再应用本步骤的 `codeSnippet`。
4. **重置行为**：点击"重置"只重置当前步骤涉及的文件到本步骤的 `codeSnippet` 初始值，不影响其他未在本步骤声明的文件，也不触发页面刷新、不影响引导进度。
5. **waitFor 判定边界**：沙箱只负责上报"代码内容是否变化 / 变化后是否命中步骤声明的校验规则（如包含某关键字符串、简单正则匹配）"，不做真正的代码执行结果断言（例如不判断"渲染出来的 DOM 是否符合预期"）；复杂断言留待未来版本迭代，当前版本 `waitFor: input` 仅做文本级匹配。
6. **语言模式与依赖锁定**：`sandboxTemplate` 与 `dependencies` 由 TutorialPackage/步骤配置决定，运行时沙箱组件不允许学习者切换 template（不提供"切换成 Vue 语法"这类入口），保证与教程内容严格对应。
7. **移动端交互**：≤768px 断点下，默认展示"讲解"Tab，学习者需手动切到"代码"Tab 才能看到沙箱；沙箱内多文件 Tab 与讲解内容的 Tab 是两层独立的 Tab 结构，不做嵌套折叠以免层级过深。
8. **主题联动优先级**：沙箱主题跟随平台明暗模式设置，学习者无法为沙箱单独设置一套独立主题（不引入" 沙箱内单独主题开关"这种非核心功能，遵循简洁守则）。
9. **元素/文件缺失降级**：若步骤配置的 `codeSnippet.file` 路径在当前文件列表里不存在，沙箱自动新建该文件并追加到文件 Tab 列表，不报错阻断；若整个 `codeSnippet` 缺失或格式非法，沙箱保留上一步骤状态不变，并在控制台记录一条告警（不影响学习者可见的 UI）。
10. **游客与登录态无差异**：沙箱运行能力本身不区分游客/登录用户，登录态只影响"进度是否被记录并跨端同步"，不影响 Run/编辑/重置的可用性。

## 4. 页面清单

| 页面 key | 页面名称 | 职责一句话 |
|---|---|---|
| sandbox-panel | 沙箱主面板（桌面右栏） | 承载多文件 Tab + 编辑区 + Run 按钮 + 预览区的一体化沙箱视图，桌面 ≥768px 下与讲解区左右并排 |
| sandbox-mobile-tab | 移动端沙箱 Tab 视图 | ≤768px 下作为讲解/代码两个 Tab 之一，全宽承载沙箱主面板的精简版布局 |
| sandbox-error-state | 沙箱异常态视图 | 展示编译报错堆栈、文件缺失降级提示、依赖加载失败提示等异常反馈，嵌入在主面板预览区内而非独立跳转页面 |

> 说明：代码沙箱模块本身页面数量少，因为它是嵌入教程详情页的一个区域组件，而非独立路由页面。上表的 3 个"页面"实际是同一沙箱组件在不同容器/状态下的呈现，为便于设计与走查单独列出。

## 5. 关键数据字段

### TutorialPackage（教程包，沙箱相关字段）
- `sandboxTemplate`: string，枚举 `react` | `vanilla` | `vue`（预留）
- `defaultFiles`: `{ [filePath: string]: string }`，教程包全局默认代码模板
- `defaultDependencies`: `{ [pkgName: string]: string }`，全局默认依赖及版本

### Step（步骤，沙箱相关字段）
- `codeSnippet`: 
  - `files`: `{ [filePath: string]: string }`，本步骤要写入/覆盖的文件内容（增量）
  - `activeFile`: string，本步骤进入时默认高亮显示的文件 Tab
  - `resetAllFiles`: boolean，是否先重置所有文件到全局默认再应用本步骤内容
- `dependencies`: `{ [pkgName: string]: string }`（可选，步骤级追加依赖）
- `autorun`: boolean，是否进入即自动 Run 一次
- `waitFor`: `"click" | "input" | "timer" | "none"`
- `waitForConfig`（当 `waitFor === "input"` 时）：
  - `matchFile`: string，需要匹配的目标文件
  - `matchPattern`: string（正则或子串），判定"用户已完成输入"的规则

### 沙箱运行时状态（前端内部，不持久化到后端，除非涉及进度记录）
- `currentFiles`: `{ [filePath: string]: string }`，当前编辑区实际内容
- `activeFile`: string，当前选中 Tab
- `isRunning`: boolean
- `hasError`: boolean / `errorMessage`: string
- `theme`: `"light" | "dark"`
- `lastRunTimestamp`: number

## 6. 异常 / 边界

| 场景 | 处理方式 |
|---|---|
| `codeSnippet` 中引用的文件路径此前不存在 | 自动新建文件并加入 Tab 列表，不报错、不阻断 |
| `codeSnippet` 缺失或字段格式非法 | 保留上一步骤沙箱状态不变，控制台告警，不弹出用户可见错误 |
| Sandpack bundler 编译失败（学习者代码有语法/运行时错误） | 预览区展示 Sandpack 原生错误堆栈，不吞错、不静默失败；Run 按钮保持可再次点击 |
| 依赖包加载失败（网络问题 / 包名不存在） | 预览区展示依赖加载失败提示，编辑区仍可正常编辑，不影响其他已加载文件的可读性 |
| 移动端窄屏（375px）多文件 Tab 过多导致横向溢出 | 文件 Tab 区域单独可横向滚动（`overflow-x: auto`），不引发整页横向滚动 |
| 学习者快速连续点击 Run | 对 Run 做防抖/禁用态处理，进行中不重复触发新的打包任务 |
| 学习者在沙箱编辑代码后，未点 Run 就切换到下一步骤 | 未运行的编辑内容不影响下一步骤的注入逻辑（下一步 `codeSnippet` 按规则覆盖或增量写入），未保存的临时修改随步骤切换丢弃，不做自动保存草稿（非核心功能，暂不实现） |
| 断网/离线状态下进入沙箱 | Sandpack bundler 依赖网络加载依赖包，离线时预览区展示"网络异常，无法加载运行环境"提示，编辑区仍可用于查看/编辑代码文本 |
| `waitFor: input` 但学习者输入内容始终不匹配 `matchPattern` | 步骤保持阻塞态（引擎侧行为），沙箱侧不做强制修正或提示"答案错误"，只如实上报匹配结果，提示文案由讲解内容/引擎步骤提示条负责 |
| 极端超长代码文件（如粘贴大段无关代码） | 不做字数上限硬限制（内部工具场景无需防滥用），但编辑区渲染依赖 CodeMirror 自身性能，不额外优化 |
