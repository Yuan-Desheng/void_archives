# 功能定义：课程讲解与步骤导航（lesson-reader）

## 1. 模块概述

`lesson-reader` 是「交互式技术教程平台」的核心承载模块，负责将 `TutorialPackage` 中的章节/步骤内容渲染为可读、可练的学习界面。它是内容与引擎分离架构下**引擎侧**最重要的消费端：接收一份结构化的教程包数据，输出「左侧讲解 + 右侧沙箱 + 顶部/侧边导航」的沉浸式学习体验，并驱动引导引擎（聚光灯高亮、等待用户操作、自动推进）完成"边读边练"的闭环。

模块自身不关心某个具体主题（React/Vue/JS/算法）的内容差异，只消费 `TutorialPackage` 定义的通用 Schema（章节、步骤、讲解内容、代码片段、高亮目标、等待条件）。新增主题只需产出符合 Schema 的新教程包，无需改动本模块任何渲染/导航/引导逻辑。

**范围边界**：本模块负责"讲解内容渲染 + 步骤导航 + 引导编排（聚光灯/等待）+ 代码沙箱承载 + 学习进度记录与展示"；教程包的编写/编辑器不在本模块范围（属教程作者工具，另立模块）；用户账户注册登录本身不在本模块范围（本模块只消费账户态、读写进度）。

## 2. 用户故事

### 学习者（游客）
- 作为游客，我希望无需登录就能选择一个教程包并开始学习，快速判断这个教程是否值得我投入时间。
- 作为游客，我希望在讲解区域看到清晰排版的文字、代码块、图片，代码块要能一眼看出关键行（语法高亮）。
- 作为游客，我希望点击"下一步"或完成页面要求的操作（比如点一下某个按钮）后，教程能自动帮我翻到下一步，而不是要我自己找入口。
- 作为游客，我希望看到"我在第几步 / 总共几步"，心里有数还要学多久。
- 作为游客，我希望在沙箱里能真的改代码、点 Run，看到运行结果，而不是一段只能看不能碰的代码截图。
- 作为游客，我希望教程告诉我"点这里"的时候，界面上真的能清楚看到该点哪里（聚光灯高亮），不用自己满屏找。
- 作为游客，我希望我的学习进度不会因为我没登录就完全丢失（至少同一浏览器内断点续学）；但我也理解要跨设备同步就得登录。

### 学习者（登录用户）
- 作为登录用户，我希望换台电脑打开同一个教程，能直接从上次学到的地方继续，不用重新翻找。
- 作为登录用户，我希望在"我的学习进度"页看到每个教程包的完成度，知道自己学了多少、还剩多少。
- 作为登录用户，我希望已经完成的步骤有明显标记，正在学的步骤有明显区分，未解锁的步骤能看到但点不进去（或有提示）。
- 作为登录用户，我希望章节目录树能直接跳到任意"已解锁"的步骤，不用每次都从头一步步点下去。
- 作为登录用户，我希望切换深色模式后，代码高亮和聚光灯遮罩的视觉效果依然清晰、不违和。

### 教程作者 / Owner
- 作为教程作者，我希望我写的 `TutorialPackage`（章节结构、讲解 Markdown/MDX、selector、waitFor 条件、codeSnippet）能被引擎正确渲染和执行，不用为了适配引擎再改一遍内容格式。
- 作为教程作者，我希望某个步骤配置的 `selector` 在当前页面找不到对应元素时，引擎能优雅降级（跳过高亮但不阻断学习流程），而不是整页崩溃。
- 作为 Owner，我希望能看到教程包整体的章节数、步骤数、完成度统计入口，用于评估内容是否覆盖了 62 篇笔记的核心知识点。

## 3. 功能点清单

| 编号 | 功能点 | 说明 |
|---|---|---|
| F1 | 教程包选择与切换 | 展示可选教程包列表（标题/描述/章节数/完成度），支持切换到另一个教程包，游客与登录用户均可浏览 |
| F2 | 章节/步骤目录树 | 树状展示章节→步骤层级，标记当前/已完成/已解锁/未解锁状态，点击已解锁步骤可跳转 |
| F3 | 讲解内容渲染 | 渲染当前步骤的 MDX/Markdown 正文，支持标题、列表、图片、代码块语法高亮、内嵌提示/警告块 |
| F4 | 上一步/下一步导航 | 步骤间线性导航，按钮态随"是否可前进/可后退/是否被 waitFor 阻塞"变化 |
| F5 | 进度指示 | 当前步/总步数、当前章节进度条、整体教程完成度 |
| F6 | selector 高亮定位 | 按步骤配置的 CSS selector 在页面中定位目标 UI 元素并框选提示；找不到元素时降级为不高亮，不阻断流程 |
| F7 | 聚光灯遮罩 | 全屏半透明模糊遮罩，挖空目标元素区域使其清晰可见；监听 resize/滚动重新计算遮罩位置 |
| F8 | 等待用户操作机制 | 支持 waitFor 类型：click（点击目标元素）/ input（目标输入框有值变更）/ timer（定时器到时）/ none（无需等待，允许直接下一步）；未满足条件时下一步按钮禁用/隐藏，满足后自动推进或高亮"下一步"可点 |
| F9 | 代码沙箱（Sandpack） | 多文件树切换、代码编辑、Run 执行、实时预览面板；语言/模板随主题（React/Vue/JS）切换 |
| F10 | 步骤代码片段注入 | 每步的 `codeSnippet` 注入沙箱指定文件；提供"重置为默认代码"入口恢复模板初始状态 |
| F11 | 响应式三档布局 | 375px（移动端讲解/沙箱可切换单栏）、768px（可切换或上下分栏）、1440px（左右分栏）三档下均无横向滚动、无重叠 |
| F12 | 明暗主题切换 | 视觉主题跟随系统或手动切换，讲解区、代码高亮、聚光灯遮罩在两种模式下均清晰可辨 |
| F13 | 登录入口与用户态展示 | 未登录展示登录入口；登录后展示用户标识、进入"我的学习进度"入口 |
| F14 | 我的学习进度页 | 列出用户已开始学习的教程包，各自完成度、最后学习时间、断点续学入口 |
| F15 | 学习进度记录与断点续学 | 步骤完成即记录（游客本地存储，登录用户服务端存储）；重新进入教程包自动定位到"上次学习位置" |
| F16 | 跨端进度同步 | 登录用户的进度以服务端为准，多端登录后进度一致；游客登录后本地进度合并至账户（如有冲突，服务端策略在业务规则中定义） |

## 4. 业务规则

### 4.1 步骤解锁与导航
- 步骤默认按教程包配置的线性顺序解锁：完成第 N 步才解锁第 N+1 步；未解锁步骤在目录树中可见但不可点击跳转（灰态 + 锁形图标或类似视觉提示）。
- "完成"判定：无 waitFor（none）的步骤，点击"下一步"即视为完成；有 waitFor 的步骤，必须先满足等待条件，"下一步"按钮才可用。
- 已完成步骤允许自由回看（回看不影响已解锁进度，也不重置完成状态）。
- 章节完成率 = 该章节内已完成步骤数 / 该章节总步骤数；教程包完成度 = 全部已完成步骤数 / 全部步骤数。

### 4.2 waitFor 等待机制
- `click`：监听目标 selector 元素的点击事件，触发后判定完成。
- `input`：监听目标 selector 输入元素的值变化（非空/变更即可，具体校验规则由教程包配置，模块只负责监听与转发判定结果），触发后判定完成。
- `timer`：配置固定等待时长（如"阅读 N 秒"），计时结束自动判定完成，UI 可显示倒计时或进度条。
- `none`：无需等待，进入步骤即视为"下一步"可用。
- 未满足等待条件时，"下一步"按钮呈禁用态，并给出简短提示（例如"请先完成上方操作"）；不允许绕过（不提供强制跳过入口，除非教程包显式标记该步骤为可跳过）。

### 4.3 selector 高亮降级
- 引擎按 selector 在当前 DOM 查询目标元素；查询成功则渲染聚光灯遮罩 + 高亮框。
- 查询失败（元素不存在/尚未渲染完成）时：不显示聚光灯遮罩，不阻断步骤流转，讲解内容正常展示；可在讲解区给出文字兜底说明（教程包内容自身负责，引擎不强制生成占位文案）。
- 目标元素在页面中位置随窗口变化（resize、滚动、内容异步渲染撑高）时，聚光灯遮罩需重新计算并重绘，避免"框选位置与实际元素错位"。

### 4.4 代码沙箱与代码片段
- 每个步骤的 `codeSnippet` 定义时，进入该步骤时自动将代码注入到沙箱对应文件并覆盖用户在该步骤内此前的编辑（离开步骤前若用户有未确认的修改，是否保留由具体交互设计决定，默认离开步骤即丢弃当前步骤内的临时编辑）。
- 沙箱运行环境（模板/依赖/预置文件）随教程包所属主题切换（React 主题用 React 模板，Vue 主题用 Vue 模板等），由 `TutorialPackage` 配置驱动，引擎不写死某一主题的沙箱配置。
- "重置为默认代码"仅重置当前步骤的代码文件为该步骤 `codeSnippet` 初始值，不影响其他步骤/文件。

### 4.5 进度存储与同步
- 游客身份：学习进度存储在本地（如 localStorage），仅在当前浏览器内有效，清除浏览器数据将丢失进度。
- 登录用户：学习进度以服务端存储为准，任意步骤完成即触发进度同步；多端登录时读取服务端最新进度。
- 游客登录转化：游客在未登录状态下产生的本地进度，登录后应尝试合并到账户进度（同一教程包取"更靠后的步骤"为准，避免进度倒退）。
- 断点续学：重新打开某教程包时，默认跳转到该教程包"最后一次学习到的步骤"，而非总是从第一步开始；用户可通过目录树手动跳转到其他已解锁步骤。

### 4.6 响应式与可用性
- 三档断点（375 / 768 / 1440）下页面不得出现横向滚动条，不得出现内容重叠遮挡。
- 移动端（375px）讲解内容与代码沙箱不同屏并列展示，提供切换入口（如 Tab 或滑动切换），默认展示讲解内容。
- 平板（768px）视具体设计决定左右分栏或上下分栏 + 切换，但需保证聚光灯高亮的目标元素在当前可视区域内可寻得（若目标元素在被切换隐藏的一侧，需有引导用户切换过去的提示，避免"等待用户操作"永远无法达成）。

### 4.7 主题与视觉
- 明暗模式切换需覆盖：讲解区背景/文字、代码块语法高亮配色、聚光灯遮罩透明度与对比度、导航与进度指示控件。
- 视觉主题参考 Vue3 官方互动教程与 aura.build docs 的克制、专业调性，不引入与教程内容无关的装饰性元素。

## 5. 页面清单

| 页面 key | 页面名称 | 职责 |
|---|---|---|
| catalog | 教程包目录 | 展示所有可选教程包（标题/描述/章节数/完成度），作为学习入口 |
| lesson | 课程学习页 | 核心页面：左侧讲解内容 + 右侧代码沙箱 + 步骤导航/进度指示 + 聚光灯引导，承载完整学习闭环 |
| my-progress | 我的学习进度 | 登录用户查看各教程包的学习进度、断点续学入口（仅登录用户可访问，游客展示登录引导） |

> 说明：登录/注册表单本身不在本模块范围内，`lesson` 与 `my-progress` 页面中只承载"登录入口/用户态展示"这一交互触点，具体登录流程由账户体系模块负责。

## 6. 关键数据字段

### 6.1 TutorialPackage（教程包）
| 字段 | 说明 |
|---|---|
| packageId | 教程包唯一标识 |
| title | 标题 |
| description | 描述 |
| topic | 所属主题（react / vue / javascript / algorithm，用于驱动沙箱模板与语言高亮） |
| chapters[] | 章节列表 |
| totalStepCount | 教程包总步骤数（可由 chapters 派生） |

### 6.2 Chapter（章节）
| 字段 | 说明 |
|---|---|
| chapterId | 章节唯一标识 |
| title | 章节标题 |
| order | 章节顺序 |
| steps[] | 步骤列表 |

### 6.3 Step（步骤）
| 字段 | 说明 |
|---|---|
| stepId | 步骤唯一标识 |
| order | 步骤在章节内的顺序 |
| content | 讲解正文（MDX/Markdown 源） |
| selector | 高亮目标 CSS 选择器（可为空，表示本步骤无高亮） |
| waitFor | 等待类型：click / input / timer / none |
| waitForTarget | 等待动作的目标（selector 或计时秒数，随 waitFor 类型而定） |
| codeSnippet | 注入沙箱的代码片段（含目标文件路径 + 代码内容） |
| skippable | 是否允许跳过等待（默认 false） |

### 6.4 学习进度（LearningProgress）
| 字段 | 说明 |
|---|---|
| userId / anonymousId | 登录用户 ID 或游客本地匿名标识 |
| packageId | 对应教程包 |
| completedStepIds[] | 已完成步骤集合 |
| lastVisitedStepId | 最后学习到的步骤，用于断点续学 |
| chapterCompletionRate | 各章节完成率（派生字段） |
| overallCompletionRate | 教程包整体完成率（派生字段） |
| updatedAt | 最后更新时间，用于跨端同步冲突判断 |

### 6.5 沙箱状态（SandboxState，运行时，非持久化核心数据）
| 字段 | 说明 |
|---|---|
| activeFile | 当前编辑中的文件路径 |
| files{} | 各文件当前内容（含用户临时编辑） |
| template | 沙箱运行模板（随 topic 决定） |

## 7. 异常 / 边界

| 场景 | 处理方式 |
|---|---|
| selector 在当前 DOM 找不到目标元素 | 不渲染聚光灯遮罩，不阻断步骤流转，讲解内容照常展示（见 4.3） |
| waitFor 目标元素在移动端被切换隐藏 | 提供切换回该视图的提示，避免用户操作条件永远无法满足 |
| 用户直接通过 URL/深链跳到未解锁的步骤 | 拒绝跳转，重定向到该教程包"最后已解锁的步骤"，并给出提示 |
| 游客本地进度与登录后服务端进度冲突 | 取步骤顺序更靠后的记录为准，避免进度倒退（见 4.5） |
| 教程包不存在或已下线 | 目录页不展示；直接访问该教程包链接时展示"教程不存在"空态，并提供返回目录入口 |
| 沙箱运行环境加载失败/超时 | 讲解内容正常展示，沙箱区域展示加载失败提示 + 重试入口，不阻断阅读 |
| codeSnippet 与沙箱当前模板文件路径不匹配 | 视为配置异常，不崩溃，沙箱区域给出"该步骤代码片段配置有误"的提示（面向教程作者的调试信息，而非面向学习者的干扰信息，具体呈现方式由设计决定） |
| timer 类型 waitFor 配置了 0 或负值 | 等价于 none，立即判定完成 |
| 网络断开导致登录用户进度无法同步到服务端 | 本地暂存待同步，恢复网络后重试同步，同步前不丢失本地已产生的完成记录 |
| 章节/步骤为空（教程包内容尚未编写完整） | 目录树正常渲染已有内容，空章节可置灰或隐藏，不影响其余章节的学习 |
| resize 过程中聚光灯遮罩计算滞后 | 遮罩位置需在 resize 结束后重新计算，避免长时间停留在错位状态（允许短暂的过渡态，但需收敛） |
| 用户在等待状态下切换离开当前步骤（如通过目录树跳转） | 允许跳转（若目标步骤已解锁），当前步骤的等待状态不视为完成，也不阻止跳转 |
